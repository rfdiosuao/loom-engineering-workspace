生产操作文档

              
这里记录后台的正确用法、字段含义和上线注意事项，避免生成授权时配置错位。

            

            生产操作提示
          

          

            

              

                
1. 生成授权

                
先保存管理员 Token，再选择继承模板。客户名、到期日期、激活次数确认后生成授权码。

                
    
                  
  • 授权码生成后只会完整显示一次，后续列表主要显示后八位。
    • 
                  
  • 客户换机器时优先查看激活记录，确认是否需要解绑。
    • 
                  
  • 批量生成前先用 1 个测试码验证客户端能正常拉取会员配置。
    • 
                

              

              

                
2. 会员套餐

                
继承模板只管功能、有效期和默认网关配置。停用后不能被新授权继承，但历史授权仍保留原配置。

                
    
                  
  • 停用模板后可以重新启用，不需要新建同名模板。
    • 
                  
  • 套餐标识建议固定为 monthly、quarterly、yearly、vip_monthly。
    • 
                  
  • 功能列表用勾选方式维护，必要时仍可手动补充自定义功能。
    • 
                

              

              

                
3. 网关配置

                
通用 LLM、AI 生图、AI 视频可以分别配置 URL、Key 和模型。图像/视频留空时继承通用配置。

                
    
                  
  • Base URL 建议统一带到 /v1，避免客户端拼接错误。
    • 
                  
  • 生图和视频如果使用不同供应商，必须单独填写对应 URL 和 Key。
    • 
                  
  • 已激活客户刷新会员状态后才会拿到最新配置。
    • 
                

              

              

                
4. 配额归属

                
配额已从授权后台移除，避免两套后台同时配置导致不一致。

                
    
                  
  • 图片、视频次数配额在 API 网关后台配置。
    • 
                  
  • 授权后台只负责告诉客户端可用功能和该走哪个网关。
    • 
                  
  • 如果未来要展示用量，建议走只读接口，不在这里写配额。
    • 
                

              

              

                
5. 审计与回滚

                
每次生成、编辑、停用、清空都会自动备份 license.db，并记录审计日志。

                
    
                  
  • 线上更新前必须备份 admin_console.html、server.py 和 license.db。
    • 
                  
  • 不要覆盖 private_key.b64，否则旧授权签名会失效。
    • 
                  
  • 误操作后优先按审计日志定位备份文件。
    • 
                

              

              

                
6. 常见故障

                
客户端看不到云端模型，通常是授权未刷新、模板停用、Base URL/Key 缺失或模型名填错。

                
    
                  
  • 后台列表确认授权码启用且未过期。
    • 
                  
  • 详情里确认客户激活次数没有超限。
    • 
                  
  • 网关配置变更后让客户在启动器里刷新会员状态。
    • 
                

              

              

                
7. 发卡网站

                
发卡网站配置只负责客户端跳转入口，不参与授权验签和套餐下发。

                
    
                  
  • 填写客户购买页，不要填写后台管理页。
    • 
                  
  • 更换发卡平台时只改这里，客户端无需重新打包。
    • 
                  
  • 如果隐藏入口，客户仍可手动输入已有授权码。
    •
