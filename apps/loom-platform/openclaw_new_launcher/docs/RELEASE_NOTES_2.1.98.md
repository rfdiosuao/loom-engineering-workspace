# LOOM 2.1.98 更新说明

## Agent 会话记忆

- 修复已保存的历史消息未转换为模型对话格式的问题。此前界面能显示历史，但模型实际只能看到当前输入。
- 同一会话现在会按用户与助手的真实顺序恢复最近上下文，可以正确回答“我上一句说了什么”等连续对话问题。
- 工具调用、审批卡片和界面状态不会混入自然语言上下文，避免干扰模型判断。
- 本次提供的是同一会话内的短期记忆；跨会话用户偏好、长期事实和语义检索记忆仍属于后续能力。

## Agent 生图与生视频

- 生成完成后，图片和视频会直接显示在 Agent 对话中，不再只保留一条工具调用记录。
- 工具完成后会自动收敛为媒体结果，避免“回复已完成但调用工具仍一直显示执行中”。
- 媒体卡片会显示本地文件名，以及传送到所选手机相册的真实状态。
- 图片、视频继续统一复用 LOOM 内置媒体任务、素材库和手机上传链路。

## 创作与媒体可靠性

- 图片接口现在校验 PNG、JPEG、WebP 文件签名，Provider 返回错误页或无效数据时不再保存成损坏图片。
- 图片生成失败与视频一致返回结构化中文原因，包括鉴权失败、接口不匹配、限流、服务不可用、超时和无效文件。
- 创作页图片与视频预览新增加载失败兜底；预览失败时明确提示文件仍保存在本地，不再显示空白区域。
- 图片和视频生成成功后继续自动写入本地素材库，并尝试传送到当前所选手机相册。

## 模型账号状态

- Agent 顶部状态现在执行真实凭据刷新，不再把只能列出模型但无法执行任务的旧登录状态显示为“模型已就绪”。
- 旧凭据缺少新版 Launcher 权限时，会提示重新登录模型账号，而不是笼统显示“任务暂未完成”。
- 已有会话和历史能力 ID 无需迁移；遇到重新登录提示时，在“模型账号”中退出并重新登录一次即可。

## 验证边界

- 本版本已通过会话历史转换、模型客户端、Agent 运行时、图片文件校验、媒体任务、附件事件、对话预览、创作页合同和 TypeScript 类型检查。
- 当前机器上的旧模型会话需要重新登录后，才能执行真实 Provider 生图/生视频端到端验证；本说明不把该外部账号步骤标记为已完成。
