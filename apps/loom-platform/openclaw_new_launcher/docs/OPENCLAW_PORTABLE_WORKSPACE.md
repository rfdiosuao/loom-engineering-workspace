# OpenClaw Portable Workspace

This document describes the merchant-neutral workspace that is bundled with the portable launcher and injected into OpenClaw at runtime.

## Goal

OpenClaw should understand the launcher before it starts working:

- it is running from a portable package that may live on a USB drive;
- generated images can be saved locally under `data/generated-images`;
- when a phone agent is configured, generated images can be imported into the phone gallery;
- phone control should prefer the high-level Agent task entry instead of raw tap/swipe tools;
- runtime context must never expose API keys, phone tokens, or customer private files.

The workspace intentionally avoids merchant-specific branding. Customer-facing theme, logo, and edition are handled by the launcher brand profile, not by the OpenClaw bootstrap files.

## Bundled Files

The source template lives in:

```text
openclaw-workspace/
```

At package/runtime creation it is copied to:

```text
data/.openclaw/workspace/
```

Important files:

| File | Purpose |
| --- | --- |
| `AGENTS.md` | Main operating instructions for OpenClaw inside the launcher |
| `SOUL.md` | Product feel, tone, and safety boundaries |
| `TOOLS.md` | Launcher-provided tool map |
| `CAPABILITIES.md` | Stable capability list and expected behavior |
| `runtime-context.json` | Current runtime facts, generated without secrets |
| `skills/openclaw-image-to-phone/SKILL.md` | Image generation and phone-gallery import flow |
| `skills/openclaw-phone-agent/SKILL.md` | Phone Agent task delegation rules |
| `skills/openclaw-portable-runtime/SKILL.md` | Portable-path and USB runtime rules |

## Runtime Context

`runtime-context.json` is regenerated by the launcher and by:

```powershell
npm run openclaw:context -- --write --json
```

It may include:

- launcher version and runtime mode;
- workspace path;
- generated image output directory;
- phone base URL and connection status;
- whether a phone token exists.

It must not include:

- raw phone token;
- API keys;
- license files;
- user private documents.

## Image-To-Phone Flow

The preferred flow is:

1. Generate or save the image under `data/generated-images`.
2. If phone import is configured, upload through the launcher API or `npm run phone:image`.
3. Use the configured album from runtime context. If missing, use `OpenClaw`.
4. Report both the local file path and the phone-side gallery path.

The launcher image page exposes this as a "send generated image to phone gallery" option. Existing personal files are not auto-uploaded.

## Packaging Rules

The portable package must include:

- `data/.openclaw/workspace/*`;
- `data/.openclaw/openclaw.json` pointing agents to `data/.openclaw/workspace`;
- `data/launcher_runtime.json`;
- `releases/agent-phone/AgentPhone_latest.apk`;
- neutral runtime context without secrets.
- only the neutral default theme and the selected delivery theme; unrelated merchant themes should not be bundled.

`scripts/verify-release.ps1` enforces these files before a package is considered deliverable.
