# LOOM 2.1.89 更新说明

## 稳定性与安全

- 手机连接令牌和配对密钥改为使用 Windows DPAPI 加密后落盘；旧版明文配置会在首次读取时自动迁移。
- Bridge 系统信息新增明确的 API 契约版本、启动器版本和能力清单，便于 Codex、CLI 与桌面端在调用前确认兼容性。
- 模板云端上传改为显式同意后启用；未确认时只在本地沉淀，不会自动上传。

## 飞书线索沉淀

- 飞书写入成功后增加远端回读校验，只有字段真实存在且一致时才标记为已同步。
- 回读暂时失败的记录进入待核验队列，不再把“命令返回成功”误当成“数据已经落表”。
- 新增 `integration feishu reconcile`，可重新核验待确认记录。
- 线索写入前按稳定业务键查询；已有记录执行更新，重复键阻断写入并提示人工合并。
- 稳定业务键兼容现有 `logId/taskLogId`，历史调用和测试写入不会因升级换键。
- 写入结果缺少 `recordId` 时会先反查远端记录，避免网络抖动后的重试重复建行。
- 相同线索并发导入现在只生成一组线索、客户和草稿，并且只执行一次飞书写入。
- 待同步重试保留原业务键和完整字段；单条异常不会清空整批队列；同一 Bridge 内并发同步同一业务键时只创建一条远端记录。
- `lark-cli` 超时或异常会降级为本地待同步，不再让线索导入整次失败。

## 安装与发布

- 修复内置 Python 的模块搜索路径，使 Bridge 的 `core.*` 导入和标准 `python.*` 测试导入在安装包运行时同时可用。
- 内置运行时构建现在会验证两种导入方式，避免开发机通过、客户电脑启动失败。
- GitHub CI 覆盖 `codex/workspace-baseline-*` 目标分支，PR 不再静默跳过检查。
- 更新交接显式终止 LOOM 管理的 Bridge 进程树，并继续扫描安装目录进程，降低覆盖安装时的文件占用失败。
- GitHub 官方 Actions 已升级到 Node 24 兼容主版本。
- 授权服务 HTTP 测试按用例复用服务器，避免 Windows CI 中请求线程堆积和数据库文件占用。

## 模板云端控制

- 开启：`loom template consent --enable --retry-pending --permission control --json`
- 关闭：`loom template consent --disable --permission control --json`
- 管理部署可显式设置 `LOOM_TEMPLATE_CLOUD_AUTO_UPLOAD=1`。
