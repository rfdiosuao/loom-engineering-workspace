# LOOM 2.3.11 更新说明

## 智能体执行可靠性

- 修复同一轮任务中非连续重复调用同一项控制能力时可能再次执行的问题。
- 控制手机、矩阵分发、媒体传输和发布操作返回结果中断时，不再自动重复执行。
- 对已经提交到后台的生图、生视频、素材传输和手机发布任务保留“结果待确认”状态，避免重复生成或重复发布。
- MCP、CLI 和麓鸣内置能力使用统一的副作用安全规则。

## 错误提示与恢复

- 无法确认执行结果时，界面会提示先检查手机、矩阵任务、素材库或发布记录。
- 明确发生在执行前的输入、目标和权限错误仍可安全重试。
- 控制类能力的未知异常会隐藏敏感内容，并停止自动重放。

## 测试覆盖

- 增加智能体工具重复调用、连接中断、后台任务失败和跨层自动重试测试。
- 增加前端中文错误提示与“禁止直接重试”交互测试。
