# LOOM 2.1.94 更新说明

## 更新与安装

- 修复升级时 `_up_` 目录中的 Python、Node 运行时未退出，导致新文件无法覆盖的问题。
- 更新前会关闭 Bridge，并清理安装目录内的自有进程；低内存下 CIM 不可用时自动降级处理。
- 如果进程仍未退出，安装会明确停止，不再带着文件占用继续更新。

## 品牌与账户

- 启动页改用新的麓鸣品牌视频，播放更顺滑；素材已转为兼容 WebView2 的 H.264，并内置失败兜底。
- 修复登录弹窗和已登录账户卡 Logo 破图，两处统一使用内置麓鸣 Logo。
- 删除不可用的启动器内邮箱注册，新用户统一前往网页注册。
- Agent 中央标识改为麓鸣 Logo，并补充克制的状态动效。

## 智能体

- 工具调用按一次运行合并展示，不再逐事件刷屏；成功后收起，失败时保留重试说明。
- 修复回复完成后仍停在“正在调用工具”的状态错乱。
- 思考、回复、工具执行和审批使用同一条运行状态链，切换更准确。
- 麓鸣手机、矩阵、生图、生视频和发布能力继续由系统提示词自主选择，不再要求用户手工勾选英文能力。
- 手机发布完整保留平台、素材、设备、正文和草稿模式；默认只允许保存草稿。
- 修复手机端明确返回“任务无法完成/平台未登录”时仍被写成成功的问题；现在 Agent 会保留真实失败原因并提示完成登录。

## 创作与手机

- 生图、生视频结果进入共享素材库，并可自动传入指定手机相册。
- 创作页补齐图片与视频预览；没有本地素材时显示明确状态。
- 大媒体改用签名流式上传，避免 base64/JSON 大包失败；旧手机仅保留安全的小文件兼容路径。
- 逐设备上传和发布失败会真实返回，不再显示假成功。
