# LOOM 2.1.95 更新说明

## 智能体执行闭环

- 修复模型工具调用漏掉必填字段时，运行在审批前直接失败的问题。
- 参数校验失败现在由 Harness 在后台发起一次受限自纠，不执行工具、不提前审批，也不在聊天区生成重复失败卡片。
- 修复模型收到校验错误后直接文字收尾、却把任务标记为完成的问题；自纠轮继续锁定原能力，只有补齐参数后才能进入审批。
- 自纠仍失败时只显示一次真实失败，不会无限重试或刷屏。
- 真实麓鸣模型混合验证已覆盖：首轮固定漏掉标题，第二轮补齐标题、正文、设备、媒体和草稿模式，最终只生成一张审批卡。

## 品牌与账户

- 登录弹窗和已登录账户卡继续统一使用内置麓鸣 Logo，不再依赖缺失的 `/logo.png`。
- 启动器内邮箱注册保持删除，新用户统一跳转网页注册。
- 启动页继续使用内置 H.264 品牌视频与静态封面兜底。

## 手机草稿说明

- 抖音草稿始终保持 `draftOnly=true`，不会公开发布。
- 当前 QA 模拟器已进入抖音手机号登录页；真实草稿需要用户先完成手机号和验证码登录。
- 未登录时麓鸣会返回真实失败，不会记录假成功。
