# LOOM / 麓鸣白标销售 PPT 排版蓝图

`white-label-sales-deck`

本文件是 LOOM / 麓鸣销售演示材料的排版合同。当前主线没有可直接编辑的 `.pptx` 成品，因此先把版式、内容结构、证据位和视觉规则固定下来，后续生成 PPT 或网页演示时按本蓝图复用。

## 视觉规则

- 画布：16:9，浅米白底 `#F6F3EA`，深墨绿标题 `#111815`，主强调色 `#0B6B5B`。
- 圆角：6-8px；只给单个证据、截图、指标块使用卡片，不做层层套卡。
- 字号：标题 28-36px，页内分组标题 18-22px，正文 13-16px，指标数字 36-52px。
- 文案：每页只保留一个主判断、三条以内证据；不要堆工程词。
- 资产：必须使用麓鸣 logo、启动器截图、安装器截图、手机矩阵截图、任务日志截图或真实命令输出。

## Slide 1：封面

`cover`

- 标题：麓鸣 LOOM
- 副标题：把 AI Agent、手机员工和企业流程装进一个可控工作台
- 画面：左侧品牌 logo + 产品截图；右侧一句价值主张。
- 证据位：版本号、演示日期、部署形态。

## Slide 2：痛点与成本

`pain-cost`

- 主判断：企业真正昂贵的不是软件，而是每天重复执行的人。
- 三个痛点：安装门槛高、账号模型配置乱、手机/桌面任务无法被统一调度。
- 指标结构：人工工时、返工率、培训成本。
- 版式：左侧痛点清单，右侧成本公式。

## Slide 3：AI 出现后的可替代流程

`before-after`

- 左栏：人工安装、手动配置、逐台手机操作。
- 右栏：LOOM 一键安装、统一模型、矩阵任务发布。
- 中间只放 3 条对比指标：耗时、可复用、可审计。

## Slide 4：产品总览

`operating-model`

- 中心：LOOM 本地启动器。
- 上游：中转站账号、模型、余额、订阅。
- 下游：Codex / Claude Code / opencode / OpenClaw / Hermes / APKClaw。
- 右侧：CLI / MCP 接入，Codex 作为总控大脑。

## Slide 5：安装器

`installer`

- 主判断：面向小白电脑，先补齐环境，再安装 Agent。
- 展示：前置环境、安装状态、启动、回滚、日志摘要。
- 证据位：安装器截图 + 安装日志 3 行。
- 不展示：过长路径、原始堆栈、无关工程参数。

## Slide 6：账号与订阅

`account-subscription`

- 主判断：用户不用手动理解 API Key，也能完成账号、余额、模型同步。
- 展示：账号、套餐、余额、到期、默认模型、同步按钮、订阅入口。
- 兜底：订阅页不能内嵌时，保留浏览器打开。

## Slide 7：手机矩阵工作台

`phone-matrix`

- 主判断：手机不是单机工具，而是可调度的电子员工。
- 第一屏：顶部指标、左侧 Worker Matrix、中间 Task Composer、右侧 Live Ops Stream。
- 关键状态：在线、忙碌、失败、等待人工、已完成。
- 安全边界：外发、评论、私信、视频发布必须人工确认并写审计。

## Slide 8：证据与护城河

`proof`

- 证据一：真实安装包、manifest、多源下载。
- 证据二：本地 Bridge、CLI/MCP、任务账本和审计日志。
- 证据三：手机端签名、设备注册、模板沉淀和经验引擎。
- 护城河表达：不是一个页面，而是一整套账号、安装、调度、审计、经验复用链路。

## Slide 9：适用客户

`customer-fit`

- 电商运营团队：多账号、多素材、多平台发布与监控。
- 本地生活/门店矩阵：批量内容执行、线索处理、流程复盘。
- 教培/知识付费团队：内容生产、分发、客服辅助。
- 内部自动化团队：把重复桌面和手机流程交给 Agent。

## Slide 10：成交页

`cta`

- 标题：先从一个高频重复流程开始
- 三步：安装麓鸣、登录中转站、选择模板发布任务。
- 行动按钮：预约演示 / 获取安装包 / 试跑一台手机。
- 底部：联系方式、版本号、回滚方式。
