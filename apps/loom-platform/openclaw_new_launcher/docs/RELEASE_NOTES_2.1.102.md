# LOOM 2.1.102 更新说明

## 原生智能体

- 修复同一会话重复提交和多条运行互相覆盖的问题。
- 工具调用按一次执行过程聚合，重复状态不再刷屏；回复与工具状态按真实时间顺序收口。
- 工具失败显示可执行的中文原因，避免只出现“任务未完成”。
- 默认启用弱安全门禁，常规手机与矩阵任务可连续执行；登录、验证码、支付、安全权限、平台风控和招聘最终决定仍由人工确认。

## 手机观察与矩阵

- 手机截图和读屏预算提升到 45 秒，单步观察预算提升到 30 秒，兼容响应较慢的真机。
- 矩阵截图与智能体截图使用同一超时契约，避免手机已返回画面但工作台提前判定失败。

## Agent 接入

- Codex 和 Claude Code 统一接入 `luming-phone-agent`，覆盖手机控制、矩阵、生图、生视频、素材传输、获客和招聘场景。
- 接入提示词内置云端 Skill 下载、SHA256 校验、离线兜底和 MCP 合并规则。
- 删除旧的分散 Skill，安装器仅迁移 LOOM 明确声明的旧 Skill，不影响用户自己的扩展。

## 内置 ADB

- 修复新电脑上的 Codex/Claude Code 找不到 LOOM 自带 ADB 的问题。
- `doctor --json` 返回真实 `adbPath`，并向受控 Agent 子进程注入 `LOOM_ADB` 和临时 PATH。
- 发布构建强制校验完整 Android platform-tools，缺少 ADB 或依赖 DLL 时拒绝生成残缺安装包。

## 验证

- Python 全量测试、前端平台合同、Node 手机能力合同和统一 Skill 校验通过。
- 前端生产构建通过，云端 Skill 与应用内置 Skill 哈希一致。
