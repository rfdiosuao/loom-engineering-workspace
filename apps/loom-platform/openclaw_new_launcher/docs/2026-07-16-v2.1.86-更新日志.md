# 麓鸣 2.1.86 更新日志

发布日期：2026-07-16

## 本地素材库

- 新增可持久保存的本地图片、视频素材库，统一收录页面生成、CLI 生成和手动导入的文件。
- 支持图片/视频筛选、预览、复制路径和删除；删除后会重新读取真实列表，避免分页游标残留。
- 修复同一秒生成多个文件时互相覆盖的问题。
- 修复大图片、大视频经由 Bridge 返回 base64 导致响应过大或页面卡死的问题；生成结果可直接使用本地资源路径预览。
- 修复图生图、图生视频切换时残留上一张参考图的问题，并统一常用比例与实际生成尺寸。

## 智能体安装

- 安装包内置 Node.js 运行时参与检测和安装，不再要求新电脑预先配置系统 Node/npm。
- Claude Code、opencode、OpenClaw 优先安装到麓鸣私有目录，避免旧的系统级命令抢占检测结果。
- 修复 npm 安装缓存错误使用 `.pkg` 后缀，导致 OpenClaw 把安装包当目录并反复失败的问题。
- 启动 Claude Code、opencode、OpenClaw、Hermes 前先执行真实版本探测；入口不可运行时明确停止，不再长期显示“启动中”。
- ChatGPT Codex 继续强制使用 Microsoft Store 原版安装与检测，不会回退安装旧 Codex CLI。
- Hermes 安装器已校验文件大小、SHA-256 和 Windows 数字签名，静默安装参数与外部入口检测保持一致。

## 环境补全与稳定性

- 首次环境补全失败后可在错误页直接“重试补全”，会重新执行分发层校验并启动 Bridge，无需盲目重启应用。
- 分发下载增加 HTTPS、哈希、超时和压缩包路径边界校验。
- NSIS 冒烟检查新增内置 Node 可执行验证和五个智能体组件目录验证。
- 修复矩阵紧急停止公开“已取消”后后台线程仍继续写日志的问题，降低升级时 Python/Node 文件占用概率。

## 发布链路加固

- 首次环境补全改为事务式替换：新运行时移动失败或安装标记写入失败时，会自动恢复上一份可用环境，避免留下半安装状态。
- GitHub Windows 正式发布统一使用受保护资源构建，Python 业务源码和未混淆前端不进入安装包。
- 移除正式发布流程里的旧 Codex CLI 种子注入；Codex 继续使用 Microsoft Store 的 ChatGPT Codex 原版安装入口。
- 正式安装包先对精确 NSIS 产物执行 Authenticode 签名，再生成 SHA-256；CI 冒烟目录与清理安全边界保持一致。
- 敏感信息扫描失败时只报告类型和文件位置，命中的账号密钥统一显示为 `[REDACTED]`，不再把密钥片段打印到日志。

## 发布验证

- 前端生产构建、平台契约、完整 Python 测试和 Rust 检查纳入发布门禁。
- 发布包继续使用受保护资源构建，Python 业务源码不以明文形式进入安装包。
- 安装包会分别在英文路径和中文用户路径执行安装、启动、Bridge 与组件目录冒烟检查。
