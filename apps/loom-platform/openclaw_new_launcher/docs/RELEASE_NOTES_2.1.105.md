# LOOM 2.1.105 更新说明

## 智能体安装与中文体验

- Codex、Claude Code 和 OpenClaw 安装完成后自动写入麓鸣管理的简体中文交互规则，同时保留用户已有配置。
- Codex 同时覆盖麓鸣私有运行目录和当前 Windows 用户的官方目录，修复 Microsoft Store 版 Codex 未继承 \`CODEX_HOME\` 时仍使用英文的问题。
- Claude Code 使用官方全局记忆文件 \`~/.claude/CLAUDE.md\`，OpenClaw 使用工作区 \`AGENTS.md\`；重复安装不会重复追加内容。
- 中文规则写入失败只显示警告，不会阻断组件安装或启动。

## 安装链路稳定性

- winget 临时网络或源错误使用有界重试，并固定非交互模式与官方 winget 源。
- 修复 winget 版本探测返回非零时仍被误判为安装成功的问题。
- 安装 Node.js、Git 等先决条件后立即从 Windows 注册表刷新当前进程 PATH，减少“已安装但麓鸣仍找不到”的假失败。
- 进入智能体安装页时提前执行缓存预检，更早暴露缺失依赖和环境问题。

## 多手机任务隔离

- 读屏和手机任务显式携带当前选中设备 ID，不再依赖容易变化的全局选中状态。
- 任务提交、后台轮询恢复和结果展示均绑定对应手机，修复切换手机后显示上一台设备结果的问题。
- 添加新手机时清空旧任务结果；切回已配置手机时恢复该设备自己的最近任务。

## 界面与商业模式调整

- 总览恢复当前版本号，便于远程支持和问题定位。
- 模型账户保留登录、余额、用量、套餐状态和外部账户中心，移除旧授权码激活入口。
- 智能体接入成为独立入口；创作、诊断、手机控制等页面按新的精简信息架构收口。
