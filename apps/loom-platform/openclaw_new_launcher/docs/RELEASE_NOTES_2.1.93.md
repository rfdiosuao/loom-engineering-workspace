# LOOM 2.1.93 更新说明

## 智能体执行

- 手机工具结果不再因嵌套过深被截断，智能体可以读取真实任务状态、当前应用和执行摘要。
- 行动任务必须执行、校验再汇报，避免反复查询状态后误报完成。
- MCP 的 `isError` 会进入失败路径，MCP/CLI 权限按能力策略真实下传。
- 新增麓鸣原生手机发布能力，智能体可完整携带平台、素材、设备、正文和执行预算；默认只保存草稿，最终公开发布仍需单次明确审批。
- 发布失败会真实返回失败，不再以退出码 0 伪装成功；空标题或正文也不会破坏命令参数解析。
- 审批卡片使用中文能力名和动作摘要，不再显示内部能力 ID。
- 修复会话初始加载假重连、搜索失败丢结果、终态仍显示运行中、归档会话混入列表和分页错页。
- 回复支持安全 Markdown 渲染，模型可恢复为“跟随默认模型”。

## 手机与矩阵

- 人工接管会绑定并恢复同一台手机的同一任务，租约切换失败会原子回滚。
- 矩阵终态会在 Agent 执行租约释放后写入，避免界面先显示完成、后台仍占用设备。
- 截图只走真实画面读取接口，不再重复触发控制调用或把 base64 图片写入任务账本。
- 画面刷新失败时保留旧画面并明确报错，同时冻结点击，避免用户误触旧坐标。
- 手动截图与后台轮询冲突时会排队补做，不再出现“点了没反应”。
- 手动截图完成后会准确显示“画面已刷新”，不会一直停在刷新中。
- 坐标点击、滑动和拖动改走手机视觉控制协议；返回、主页、文字输入继续复用快速控制协议。
- 手机协议暂不支持“最近任务”时显示明确中文提示，不再暴露英文底层错误。
- 删除的设备不会被旧矩阵快照重新带回；连接状态只有在实时流和快照都健康时才显示已连接。
- 手机地址或令牌修改后，保存前禁止使用旧配置检测、截图或下发任务。

## 安全与可靠性

- Bootstrap 远程和缓存 manifest 必须通过内嵌 Ed25519 公钥验签，篡改镜像、哈希或安装路径会被拒绝。
- 在线便携包保留完整签名信封，发布脚本不再剥离 `distribution` 签名覆盖。
- 登录码、授权码和绑定票据在 CLI、MCP 和任务审计中统一脱敏。
- 低虚拟内存或 `taskkill` 失败时，升级流程会降级终止进程并优先恢复用户数据、应用和注册表。
- CI 现在覆盖全部 PR 目标、前端契约、Node 手机/发布契约和 Rust 行为测试。
- 商业安装包使用 protected release 分层，业务 Python 编译后打包并进行双重源码泄漏扫描；ASCII 与中文路径安装烟测均通过。

## 交互细节

- 启动、矩阵同步、智能体思考和搜索失败都有明确状态与重试入口。
- 确认弹窗支持 Escape、焦点圈定和关闭后焦点恢复；Toast 支持辅助技术播报。
- 保持现有整体视觉结构，仅做紧凑状态、错误反馈和键盘操作优化。
