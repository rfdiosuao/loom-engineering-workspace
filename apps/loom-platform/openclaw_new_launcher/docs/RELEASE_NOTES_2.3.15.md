# LOOM 2.3.15 更新说明

## 智能体可靠性

- CLI 和 MCP 现在识别 HTTP 200 中的业务失败，不再把 `ok: false`、`sent: false` 或失败状态显示成成功。
- 智能体会话、运行领用、审批和幂等提交增加跨进程文件锁，多个执行器同时工作时不会重复调用工具或覆盖事件序号。
- 默认采用强安全策略；真实发布、矩阵全选和设备组操作必须经过明确目标与授权校验。
- 外部 Skill/MCP 能力名称增加碰撞保护，执行时仍保留原始能力 ID，避免同名转换后调用错工具。
- 模型账号的手机模型选择、Wire 候选配置验证、诊断范围和主题设置均完整传递并持久化。

## 手机矩阵

- 矩阵取消、重试和状态查询使用权威活动状态；不存在的活动返回明确失败，不再假成功。
- 取消只有在服务端确认后才停止对应本地作业，避免“取消失败但本地任务已停”。
- 不确定执行结果进入“需要人工检查”，不会自动重复下发可能仍在手机执行的任务。
- 设备失败原因在重试与智能体结果中保留，界面可直接显示真实错误。
- 成功设备同时兼容 `succeeded` 和 `completed` 状态，矩阵完成计数准确。

## 账户与集成

- 验证码发送固定使用登录用途，发送失败会返回可操作错误。
- 飞书测试写入使用高强度唯一 ID，同一秒内连续测试不会互相覆盖。
- 手机模型、事件流控制、Wire 验证参数和诊断范围已贯通 CLI、MCP 与后端。
