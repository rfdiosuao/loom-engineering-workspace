# LOOM 2.1.90 更新说明

## 麓鸣原生中枢智能体

- 智能体改为麓鸣内置原生运行时，不再要求用户安装或选择第三方 Agent。
- 麓鸣 MCP、CLI、手机矩阵、媒体生成和系统能力统一注册到智能体，由系统提示词根据任务自动选择，不再让用户手动勾选能力。
- 能力目录完成中文语义化，模型可按业务含义识别状态、模型、账号、图片、视频、手机、矩阵、定时任务和诊断能力。
- 新增完整执行检查点、工具调用记录、取消、恢复、审批和目标范围冻结，重启或断流后可以恢复真实运行状态。
- 图片生成、图片编辑、视频生成、手机截图、手机读屏、手机快速任务和多机矩阵下发均已接入同一原生 Agent 执行链。

## 智能体工作台

- Composer 重构为简洁对话入口，移除能力选择、设备编号和设备组文本栏，避免把内部协议暴露给用户。
- 新增会话级模型切换、全局默认模型设置和模型管理入口。
- 新增自动范围、全部在线设备和手动选择设备/设备组，模型无法擅自扩大用户选定范围。
- 发送按钮支持发送与停止状态切换；Enter 发送，Shift+Enter 换行。
- 新增稳定的“正在思考”反馈、中文审批卡片、矩阵进度和不暴露内部 ID 的错误摘要。
- 移除重复的新会话按钮和仅供开发调试的顶部入口，运行详情改为按需打开的不透明侧栏。
- 修复小屏下任务范围菜单选择后不收起、遮挡消息输入框的问题。

## 手机连接修复

- 修复 DPAPI 加密手机令牌在 Python Bridge 已解密、Node 手机链却再次读取密文对象导致的持续“连接失败”。
- 解密后的手机配置只通过受控子进程环境传递，不写回明文文件、不进入命令行参数、API 响应或任务结果。
- 手机状态检测现在精确绑定所选设备，并同时校验任务、设备结果和 `status.online`。
- 新鲜的显式连接检测优先于旧 Matrix 快照，避免历史离线状态覆盖当前在线结果。
- 增加令牌缺失、授权失败、手机服务不可达、超时和 daemon 异常的中文可行动提示。

## 安装与保护包

- 保护版 Python 模块支持无源码导入，避免随包 Agent bootstrap 因 loader 仅支持主程序执行而失败。
- NSIS 烟测新增麓鸣原生 Agent profile、能力注册和 bootstrap 验证。
- 安装包版本统一升级到 `2.1.90`。
