# Commercial License Paywall Design

Date: 2026-07-09
Project: 麓鸣AI矩阵获客工作台
Scope: 商业化交付 1.0，月卡授权版启动付费墙

## Summary

商业化交付 1.0 先做一个清晰、可售后、可验收的授权入口：用户打开软件后先看到品牌启动动画；如果本机没有有效授权，动画结束后停在授权码激活页；授权成功后进入主工作台。未授权状态下不进入获客、飞书、手机矩阵、工作台等核心能力，但允许用户完成授权激活、复制机器码、导出诊断信息和查看必要的环境检测状态。

这个设计不新增支付系统，不重做授权服务，不改变底层 CLI、Feishu API、手机 Agent 协议，也不扩大任何自动触达能力。

## Goals

- 让安装包具备商业交付门禁：没有授权码不能使用核心功能。
- 让客户明确知道软件不是卡死，而是在等待授权激活。
- 让售后能快速判断问题属于未授权、授权过期、设备不匹配、网络失败还是本机环境异常。
- 让月卡、体验版、团队版、VIP 等套餐能通过现有授权 payload 的 edition、plan、features、expiresAt、deviceLimit 等字段表达。
- 保留少量未授权可用能力，避免客户无法激活或无法反馈问题。

## Non-Goals

- 不做在线支付、订单系统、分销系统或自动续费。
- 不在客户端保存真实账号密码、模型 token、Gitee token、飞书敏感 token。
- 不新增自动私信、自动评论、自动加好友、自动群发能力。
- 不改 installer、证书、Gitee 上传、服务器 DNS 或生产授权服务部署，除非后续实施计划单独确认。
- 不用无限加载动画伪装未授权状态。

## Product Experience

### Authorized Launch

1. 用户打开软件。
2. 显示现有品牌启动动画。
3. 客户端检查本地授权缓存，并通过现有授权校验逻辑确认有效性。
4. 授权有效时直接进入主工作台。
5. 主界面可在账号/授权页展示客户名、套餐、到期时间和功能权限。

### Unauthorized Launch

1. 用户打开软件。
2. 显示品牌启动动画。
3. 授权缺失、过期、停用、设备不匹配或校验失败时，进入授权付费墙页面。
4. 页面明确显示需要输入授权码，不显示空白页或无限加载。
5. 用户输入授权码并点击激活。
6. 激活成功后保存签名授权缓存，并进入主工作台。

### Expired Or Invalid License

当授权过期、停用或设备不匹配时，用户回到付费墙页面。页面应显示可读原因：

- 授权已过期：展示到期日期，提示续费。
- 授权已停用：提示联系服务方。
- 设备不匹配：展示本机机器码/安装 ID，提示重新绑定或换码。
- 功能未开通：主界面内锁定对应功能并展示当前套餐未开通。
- 网络失败：保留本地有效缓存的宽限策略；没有有效缓存时停在授权页并提示稍后重试。

## Paywall Page Layout

付费墙页面使用现有品牌壳，不进入完整侧边栏工作台。页面应看起来像正式商业软件的激活页，而不是错误页。

### Header

- Logo
- 产品名：麓鸣AI矩阵获客工作台
- 副标题：手机矩阵获客、飞书线索沉淀与 AI 销售助理能力

### Activation Panel

- 授权码输入框
- 激活授权按钮
- 当前状态提示
- 机器码/安装 ID
- 复制机器码按钮
- 导出诊断按钮
- 联系开通入口

### Capability Preview

展示授权后可解锁的能力，但不使用虚假数据：

- 多台手机矩阵获客总控
- 线索发现与客户池沉淀
- AI 跟进草稿
- 飞书多维表格同步
- 模板云端沉淀
- 售后诊断与日志留痕

### Allowed Unauthorized Actions

未授权状态下只允许：

- 激活授权码
- 读取基础版本号
- 复制机器码/安装 ID
- 导出脱敏诊断包
- 查看基础环境检测结果
- 打开购买或客服链接

未授权状态下禁止：

- 进入获客工作台
- 写入飞书多维表格
- 调度手机矩阵
- 发布内容
- 触发私信、评论、加微、群发等外部动作
- 上传模板到云端
- 使用付费模型网关

## License Data Model

客户端优先复用现有授权结构，不引入新的授权协议。商业交付 1.0 只要求这些字段可被 UI 清晰展示和校验：

- licensee: 客户名或团队名
- edition: 版本名，例如 trial、standard、team、vip
- plan: 套餐名，例如 monthly、quarterly、yearly、vip_monthly
- expiresAt: 到期时间
- features: 功能权限列表
- deviceId 或 installId: 授权绑定设备标识
- deviceLimit: 允许设备数
- gatewayBaseUrl: 可选模型或网关地址
- gatewayAccessToken: 可选网关访问凭证，客户端不得明文展示

建议 feature key 初始采用：

- acquisition.workbench
- acquisition.feishu
- matrix.devices
- templates.cloud
- publishing.draft
- diagnostics.export

## Routing And Access Control

启动时增加一个授权门禁状态：

- checking: 正在检查授权，显示品牌动画或短暂检查状态。
- authorized: 授权有效，进入主应用。
- unauthorized: 无授权，进入付费墙。
- expired: 授权过期，进入付费墙并显示续费提示。
- device_mismatch: 设备不匹配，进入付费墙并显示机器码。
- offline_grace: 网络失败但本地缓存仍在宽限期内，可以进入主应用并显示离线提示。

前端路由不应只依赖 UI 隐藏。所有受保护后端 API 仍必须通过现有授权校验保护。付费墙只是第一层体验门禁，后端授权校验是实际安全边界。

## Error Handling

授权页必须避免三类体验问题：

- 不出现无限加载。
- 不吞掉错误原因。
- 不把敏感 token、账号、私密配置展示到页面或诊断包。

推荐错误文案：

- 授权码无效：授权码不存在或格式不正确，请检查后重试。
- 授权已过期：当前授权已于 YYYY-MM-DD 到期，请续费后继续使用。
- 授权已停用：当前授权已被停用，请联系服务方处理。
- 设备不匹配：当前电脑未绑定此授权，请复制机器码联系服务方。
- 网络不可用：暂时无法连接授权服务，请检查网络后重试。
- 服务异常：授权服务暂时不可用，请稍后重试或导出诊断信息。

## Diagnostics

诊断包面向售后，不面向营销。未授权也可以导出，但必须脱敏。

允许导出：

- 应用版本
- 构建版本
- 安装路径
- 系统版本
- 授权状态摘要
- 机器码/安装 ID
- Python Bridge 状态
- Node Runtime 状态
- Feishu 绑定状态摘要
- 手机矩阵状态摘要
- 最近错误日志摘要

禁止导出：

- 明文授权码
- 模型 API Key
- Feishu access token
- Gitee token
- 用户私信、客户手机号、微信号等生产客户数据
- 真实账号密码

## Commercial Packages

首版 UI 只展示，不在客户端完成订单管理。

- 体验版：7 天，适合试装和跑通配置。
- 标准版：单团队基础获客，包含获客工作台、飞书同步、诊断导出。
- 团队版：多设备矩阵、模板云端、更多设备额度。
- VIP 版：更高设备额度、定制模板、优先支持。

套餐具体价格不写死在客户端，以免后续调整需要重新发包。客户端只展示授权服务返回的 edition、plan、expiresAt 和 features。

## Verification Plan

实现后至少验证：

- 无授权首次启动进入付费墙，不进入主工作台。
- 输入有效授权码后进入主工作台。
- 授权过期显示过期原因。
- 设备不匹配显示机器码。
- 网络失败不会空白；有有效缓存时按离线宽限策略进入，无缓存时停在授权页。
- 未授权无法访问获客、飞书、手机矩阵、模板云端等核心功能。
- 授权后原有获客、飞书、手机矩阵、安装器构建流程不被破坏。
- 诊断包不包含密码、token、授权码和客户私密数据。

建议自动化检查：

- 授权页契约测试：标题、授权码输入框、激活按钮、机器码、诊断按钮存在。
- 路由门禁测试：未授权状态下主功能不渲染。
- API 授权测试：受保护接口无有效授权时被拒绝。
- 构建测试：npm run build。
- Python 测试：python -m unittest discover -s python\tests -p "test_*.py"。
- 打包测试：NSIS 构建成功并记录安装包 SHA256。

## Rollout Plan

1. 先实现客户端付费墙和路由门禁。
2. 接入现有 license current、activate、authorized 接口。
3. 增加套餐和 features 的 UI 展示。
4. 加诊断导出入口和脱敏检查。
5. 跑发布级验证并生成交付报告。
6. 再决定是否接入服务器后台的批量授权码管理、下载页、版本升级页。

## Open Decisions

- 离线宽限期长度建议默认为 72 小时，最终可由授权服务返回配置。
- 客服/购买入口建议使用远程配置下发，避免写死个人联系方式。
- 是否允许体验版默认试用，需要单独确认；本设计默认没有授权码就不进入主工作台。

## Acceptance Criteria

- 客户安装后，如果没有授权码，会看到清晰的激活页，而不是卡死、空白或无限加载。
- 已授权客户可以正常进入现有主应用，不影响获客、飞书、手机矩阵和安装器流程。
- 未授权客户无法使用核心付费能力。
- 售后可以通过机器码和脱敏诊断包定位授权与环境问题。
- 所有真实对外发送、评论、私信、加微、发布动作仍默认走草稿、人工确认、频控和日志留痕。
