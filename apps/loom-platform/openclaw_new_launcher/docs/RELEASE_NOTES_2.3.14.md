# LOOM 2.3.14 更新说明

## 智能体可靠性

- 同一运行增加持久化租约与原子领取，避免多个执行器重复调用工具。
- 审批恢复时重新校验输入、目标、风险、能力实现与策略，过期审批不再复用。
- 修复运行先显示完成、最终回复尚未写入会话的竞态。
- MCP 成功结果按真实业务载荷解析；畸形或不确定结果不再伪装成功。
- 工具历史保持完整 JSON，验证码、令牌等敏感信息继续脱敏。

## 手机矩阵与媒体

- 手机仍在执行时，超时结果标记为不确定，不再自动重复下发。
- 截图与读屏缓存按真实设备隔离，切换手机不会串图。
- 支持按活动 ID 查询展示窗口之外的旧矩阵任务。
- 生成图片或视频默认只保存本地；选择手机后才触发外部传输审批。
- 指定手机传输失败会返回真实失败，不再显示假成功。

## 会话与界面

- 终态消息、运行和工具卡片不再被旧实时事件倒退为进行中。
- 实时事件断档会从已提交游标补齐，账本读取失败会明确结束流并提示。
- 归档会先取消活动运行，再原子清理会话、消息、草稿、游标和工具状态。
- 消息重试复用同一幂等键，防止重复提交。

- 前端平台契约：173 项通过。
- 手机与矩阵路由：131 项通过。
- 智能体、媒体、MCP 整合测试：330 项通过。
