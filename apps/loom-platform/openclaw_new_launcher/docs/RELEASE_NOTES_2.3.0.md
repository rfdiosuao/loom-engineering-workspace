# LOOM 2.3.0 更新说明

## 全新更新中心

- 启动后自动检查正式版本，也可以从设置页手动打开更新中心。
- 在应用内展示版本号、发布时间、安装包大小和完整更新说明。
- 支持后台下载、实时进度、断点续传、暂停后继续和跳过指定版本。
- 下载完成后执行 SHA256 与 Windows 发布者签名双重校验，校验失败不会安装。

## 安全升级与恢复

- 更新前自动备份用户数据、应用目录和必要的注册表信息。
- 静默安装后执行新版运行时与 Bridge 健康检查。
- 新版启动失败时自动恢复上一版本，并在下次启动时显示恢复原因与处理建议。
- 更新交接只关闭当前安装目录拥有的进程，避免误伤其他 Python 或 Node 任务。

## 更新体验

- 更新下载可在后台继续，不阻塞手机矩阵、智能体和媒体创作。
- 已验证的安装包会被安全复用，应用重开后无需重复下载。
- 用户主动取消更新时保留有效下载进度，不再弹出错误提示。
- 没有更新回执时不再持续轮询，减少每次启动的无效请求。

## 稳定性

- 版本比较改为语义版本判断，不会把旧版本误报为可更新版本。
- 正式发布资产统一为 `LOOM-版本号-setup.exe`，并同时生成 SHA256 校验文件。
- 发布流程继续执行受保护构建、Authenticode 签名和双路径安装冒烟测试。
