# OpenClaw + APKClaw 组合能力清单

更新时间：2026-05-11

这份文档描述的不是 APKClaw 单独能点哪里、滑哪里，而是 OpenClaw / 启动器 / 专门 Agent / APKClaw 组合后，能承接哪些真实任务。

核心定位：

- OpenClaw：负责理解目标、拆任务、分析、写报告、生成素材、调用技能。
- 启动器：负责配置、运行时、文件落地、图片/视频素材中转、手机连接。
- 专门 Agent：负责调研、采集、视觉识别、文案、测试、演示复盘。
- APKClaw：负责手机端执行，包括观察、点击、输入、滑动、打开 App、采集结构、录屏和回传结果。

一句话：OpenClaw 是脑，APKClaw 是手机上的手。

当前协作规则补充：
- 每台新手机先做初始化体检，读取设备能力、应用清单、当前屏幕节点统计和 `vision` 建议。
- 如果 `currentScreen.nodeCount` 很低、结构树为空，或 `vision.recommended=true`，OpenClaw 应先改写给 APKClaw Agent 的任务，让它按“截图观察 + 可见入口 + 安全候选动作”方式执行，而不是继续强压无障碍节点采集。
- OpenClaw 亲自调用视觉坐标动作只作为调试、兜底或 APKClaw 多次失败后的介入；默认主线仍然是 OpenClaw 指挥 APKClaw Agent。

## 1. 电商竞品调研

适用场景：

- 淘宝、京东、拼多多、抖音商城、闲鱼等平台的商品调研。
- 查看某个关键词下的商品标题、价格区间、卖点、主图风格、评价入口、店铺信号。
- 对比多个平台同类商品的价格、包装、标题关键词、赠品、销量/评价露出。

现在可以做：

- OpenClaw 生成调研目标，例如“调研 500 元左右移动硬盘”。
- APKClaw 打开指定 App 或 Via 浏览器。
- 搜索关键词、翻页/滑动、观察页面。
- 对有无障碍节点的列表页，读取标题、价格、店铺名、位置、按钮、可点击入口。
- 对结果进行初步筛选，例如“价格在 400-650 元、看起来像移动硬盘、排除广告位”。
- 输出竞品摘要、价格带、标题关键词、用户可见卖点、下一步深挖建议。
- 必要时开启手机录屏，把调研过程拉回 PC 作为演示素材。

典型任务：

```text
调研淘宝上 500 元左右移动硬盘的竞品，采样前 20 个结果，按价格、容量、品牌、卖点做一份摘要。
```

当前边界：

- 商品列表如果是纯图片/强 WebView/强反爬页面，结构化节点可能不足，需要进入截图 + OCR/VLM 模式。
- 账号、地区、推荐流会影响结果，报告里需要标注“当前账号/当前设备可见结果”。
- 支付、下单、联系卖家等敏感动作不应默认执行。

## 2. BOSS 直聘岗位分析

适用场景：

- 分析某类岗位需求，例如“AI 产品经理”“短视频剪辑”“跨境电商运营”。
- 统计薪资范围、城市分布、经验要求、学历要求、关键词、公司类型。
- 找出岗位描述里的高频技能和隐性门槛。

现在可以做：

- APKClaw 打开 BOSS 直聘并搜索岗位。
- 使用岗位采集器读取可见岗位卡片。
- 滑动加载更多岗位，逐批采集。
- OpenClaw 对岗位进行去重、归类、薪资带分析、关键词提取。
- 输出“岗位画像”“求职建议”“招聘方需求变化”“简历优化建议”。

典型任务：

```text
在 BOSS 直聘搜索杭州 AI 产品经理，筛选 20 个岗位，分析薪资、经验、技能要求和公司类型。
```

现在的优势：

- 这是最适合当前结构化采集器的场景之一。
- 岗位卡片通常比短视频/游戏更容易通过无障碍树读取。
- 可以比较快形成可复用模板。

当前边界：

- 页面弹窗、登录状态、推荐流变化会影响采集效率。
- 如果岗位详情页内容很长，需要分页读取和摘要合并。
- 需要继续增强“采集到空结果时的恢复策略”，例如前台包校验、返回列表页、换关键词、切视觉模式。

## 3. 闲鱼好货筛选

适用场景：

- 找二手好价，例如手机、硬盘、显卡、相机、游戏机、家具。
- 初筛“价格明显偏低但不像诈骗”的商品。
- 比较标题、价格、地区、卖家描述、发布时间、交易入口。

现在可以做：

- 打开闲鱼并搜索关键词。
- 读取列表页的商品标题、价格、地区、卖家/曝光信息。
- 滑动采样多屏结果。
- OpenClaw 结合常识做风险分层：
  - 价格异常低；
  - 标题含糊；
  - 描述缺关键信息；
  - 同城/距离因素；
  - 是否需要继续点进详情核查。
- 输出候选清单和避坑提醒。

典型任务：

```text
在闲鱼找 1TB 移动硬盘好价，先筛 30 个结果，挑出 5 个值得点进去看的，并说明理由和风险。
```

当前边界：

- 不能替用户私聊、付款、砍价或提交交易。
- 判断“好货”需要结合图片细节时，要接入截图视觉识别。
- 风险判断只能作为辅助，不应承诺真假鉴定。

## 4. 评论区与口碑采集

适用场景：

- 抖音评论区、电商商品评论区、京东/淘宝/拼多多评价页。
- 收集用户抱怨点、好评关键词、差评原因、真实使用场景。
- 给商家做选品、详情页优化、广告文案、客服 FAQ。

现在可以做：

- 打开目标商品或内容页。
- 找到评论/评价入口并进入。
- 对可读取节点的评论列表进行采样。
- 滑动加载更多评论。
- OpenClaw 做情绪分类、需求归因、痛点聚类、文案提炼。

典型任务：

```text
采集这个商品评价区前 50 条用户反馈，整理出用户最在意的 10 个点，并给详情页优化建议。
```

当前边界：

- 抖音等 App 的评论区有时节点不稳定，可能需要视觉模式。
- 私信、账号昵称、头像等个人信息应脱敏。
- 当前更适合“采样分析”，不是大规模爬虫。

## 5. 小红书/抖音内容灵感调研

适用场景：

- 调研某个关键词下的爆款标题、封面卖点、评论反馈。
- 为广告视频、种草笔记、短视频脚本找方向。

现在可以做：

- 打开 App 或 Via 浏览器。
- 搜索关键词。
- 采样可见内容卡片、标题、互动露出、入口。
- 对多个结果做标题套路和内容角度总结。
- 截图/录屏回传 PC，作为后续 AI 生成脚本、分镜、封面参考。

典型任务：

```text
调研小红书上“移动硬盘推荐”的内容方向，找出 10 个高频标题角度和 5 个可复用封面卖点。
```

当前边界：

- 强视觉内容必须依赖截图 + VLM 才能稳定理解封面。
- 不应自动点赞、收藏、评论、关注。

## 6. 手机 App 竞品体验巡检

适用场景：

- 分析一个竞品 App 的首页结构、入口层级、注册/登录前体验、购买路径。
- 做产品经理视角的 UI/UX 走查。
- 记录关键流程的视频证据。

现在可以做：

- 打开指定 App。
- 获取当前页面标题、可见入口、按钮、列表结构。
- 按 OpenClaw 的计划逐步探索，例如首页、分类、搜索、详情页、设置页。
- 每一步截图、记录节点、写出观察。
- 需要演示时开启录屏，结束后把 MP4 拉回电脑。

典型任务：

```text
帮我走查这个 App 的新用户首页，记录 5 个主要入口、3 个转化路径和明显的体验问题。
```

当前边界：

- 登录、实名、支付、隐私授权等需要用户明确确认。
- 没有节点的页面需要视觉模式辅助。

## 7. AI 生图到手机相册

适用场景：

- OpenClaw 生成图标、海报、商品图、头像、封面。
- 生成后直接传到手机相册，方便在手机 App 里使用。
- 给商家演示“电脑生成素材，手机端马上可用”。

现在可以做：

- OpenClaw 调用图片生成 API。
- 启动器保存图片到 `data/generated-images/`。
- 调用 APKClaw 的图片导入接口。
- 图片进入手机 `Pictures/<album>`。
- 用户可以在微信、QQ、相册、剪辑 App、发布工具里选择图片。

典型任务：

```text
生成一张移动硬盘促销海报，保存到电脑，并直接传到手机相册，方便我发朋友圈或导入剪辑软件。
```

当前边界：

- 不自动读取用户相册内容。
- 不自动代发朋友圈、聊天消息或平台内容。

## 8. 手机录屏素材采集

适用场景：

- 收集 App 操作过程，给 AI 做演示视频、教程、复盘。
- 记录 APKClaw 执行任务失败时的现场。
- 给客户展示“AI 控手机”的真实过程。

现在可以做：

- OpenClaw 判断是否需要录屏。
- APKClaw 发起 Android 系统录屏授权。
- 用户确认后开始录屏。
- 任务结束后停止录屏。
- 启动器把 MP4 拉回 `data/phone-videos/`。
- OpenClaw 后续可以总结、剪辑、配文、生成演示说明。

典型任务：

```text
录一段 APKClaw 在京东搜索移动硬盘并筛选结果的过程，结束后把视频拉到电脑做演示素材。
```

当前边界：

- 录屏必须有用户明确意图。
- 每次开始录屏都要尊重 Android 系统授权。
- 私密 App、聊天、支付、相册、文件管理场景默认不录或需要再次确认。

## 9. 手机端 QA 与回归测试

适用场景：

- 测试 APKClaw 自己的功能是否正常。
- 测试启动器连接手机、截图、结构树、Agent 任务、录屏、图片导入。
- 测试第三方 App 的关键路径是否可走通。

现在可以做：

- 自动检查手机在线、版本、无障碍、截图、结构树、亮屏状态。
- 发起只读任务，验证不会误点。
- 发起安全操作任务，验证点击、输入、返回、滑动。
- 录屏记录失败路径。
- 输出测试报告和问题列表。

典型任务：

```text
帮我验收 APKClaw 当前版本：连接、截图、结构树、Agent 任务、录屏、视频下载都测一遍，给我问题清单。
```

当前边界：

- 第三方 App 的账号状态、弹窗、地区推荐会影响复现。
- 需要把更多验收项纳入脚本，减少手工测试。

## 10. 手机网页浏览与资料检索

适用场景：

- 使用 Via 浏览器打开网页、搜索、查看下载页。
- 手机端页面和 PC 端结果不一致时，以手机真实页面为准。
- 做移动端落地页检查。

现在可以做：

- 优先使用 Via Browser。
- 打开 URL 或搜索关键词。
- 读取页面可见内容。
- 点击页面入口、返回、滑动。
- 截图或录屏保存证据。

典型任务：

```text
用手机浏览器打开这个商品落地页，看看首屏卖点、按钮、加载情况和移动端排版问题。
```

当前边界：

- 复杂网页、Canvas、视频页需要视觉模式。
- 下载/安装 APK 这类流程容易被系统确认页打断，需要显式确认。

## 11. 游戏 / Canvas 视觉模式

适用场景：

- 洛克王国、小游戏、强 Canvas/WebView 页面、无障碍节点为空的画面。
- 做安全探索、按钮识别、状态确认、故障复盘。
- 在游戏页里完成轻量导航，例如打开设置、关闭弹窗、返回主界面。

现在可以做：

- APKClaw Agent 先观察，发现节点为空时返回 `needs_vision`，不会盲目滑动或乱点。
- OpenClaw 捕获带网格的视觉帧，识别按钮位置和大概意图。
- OpenClaw 生成一条带 `targetLabel` 和 `reason` 的视觉计划。
- 启动器安全黑名单先审查计划。
- 默认把计划交给 APKClaw Agent 以 `safe_action` 执行，而不是直接坐标遥控。
- 执行后再次截图，验证页面有没有变化。

典型任务：

```text
看看洛克王国当前页面，判断有没有安全的设置入口；不要点登录、授权、清缓存、上报日志或退出。
```

当前边界：

- 这是“视觉闭环 MVP”，还不是稳定游戏自动过关。
- 复杂战斗、血条、背包、地图、寻路需要 OCR/VLM 状态机和任务模板。
- 登录、授权、充值、购买、清缓存、上报日志、退出游戏等目标默认阻断。

## 12. 资料整理与报告生成

适用场景：

- 把手机端采集到的信息变成报告。
- 把多平台采样结果变成表格、摘要、建议。
- 给商家输出可读结论，而不是一堆原始节点。

现在可以做：

- 将采集结果按字段整理。
- 去重、分组、提炼关键词。
- 生成 Markdown 报告。
- 给出下一步采集建议。
- 把截图、录屏、本地文件路径一起归档。

典型输出：

- 竞品调研报告。
- 岗位需求分析。
- 闲鱼好货候选清单。
- 商品评论痛点总结。
- App 巡检报告。
- 手机任务失败复盘。

## 13. 当前还不应夸大的能力

这些方向有市场，但现在不应该包装成“已经稳定完成”：

- 大规模爬虫：当前更适合采样、辅助分析，不是无休止批量抓取。
- 自动交易：不自动下单、付款、转账、私聊成交。
- 微信/QQ 私密内容读取：默认保护隐私，只做结构和状态层面的安全观察。
- 游戏自动过关：APKClaw 已有游戏/视觉闭环 MVP，但复杂过关仍需要视觉识别、状态机、任务模板，目前还不是稳定商品能力。
- 完全无人值守：弹窗、登录、授权、系统确认、风控都需要人工兜底。

## 14. 最有商业价值的优先组合

建议优先产品化这几条：

1. BOSS 岗位分析助手
2. 闲鱼好货筛选助手
3. 电商竞品调研助手
4. 商品评论/口碑分析助手
5. 手机 App 巡检与录屏复盘助手
6. AI 生图到手机相册工作流
7. 手机任务演示素材采集工作流

原因：

- 它们都能体现“OpenClaw 负责脑力，APKClaw 负责手机执行”。
- 对商家、运营、求职、产品经理、内容创作者都有明确价值。
- 结果可以落成报告、表格、截图、录屏，不只是演示“AI 会点手机”。
- 对安全边界相对友好，不需要默认触碰支付和私密聊天。
