# LOOM 2.3.13 更新说明

## 智能体工具可靠性

- 修复矩阵重试可能返回假成功、缺少新任务却被智能体当作已完成的问题。
- 修复矩阵任务提交回执中断后被错误允许重放的问题，避免手机仍在执行时重复下发。
- 修复矩阵取消未校验商业授权的问题。
- 修复同一秒连续产生多个审批时，界面可能选中已经处理过的旧审批。

## 审批与外部写入

- 矩阵重试在强门禁模式下必须重新审批，审批令牌只能使用一次。
- 用户批准后由麓鸣服务端注入确认状态，模型填写 `confirmed=true` 不再等同于用户批准。
- 验证码发送、飞书写入、飞书重试同步和获客结果同步现在按对外写入处理。
- 飞书建表按创建外部资源处理；真实获客运行即使在弱门禁模式下也必须明确审批。

## 测试覆盖

- 完成 82 项智能体能力的一对一执行、风险、目标范围和审批契约审查。
- 完成 1537 项 Python 全量回归、162 项前端平台合同和 63 项 Node 手机脚本合同。
- 增加矩阵重试、未知提交结果、审批并发、外部写入风险和服务端确认注入的对抗测试。
