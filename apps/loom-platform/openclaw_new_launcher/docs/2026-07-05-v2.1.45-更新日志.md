# 更新日志 v2.1.45

日期：2026-07-05

## 手机 Agent / APKClaw 调用

- 修复 LOOM `phone quick-task` / `phone run-task` 显式运行预算未完整下传的问题：`--profile`、`--timeout-sec`、`--max-wait-sec`、`--max-rounds`、`--poll-ms` 现在会进入 `/api/phone/task` 请求体。
- 保持 safe/fast 默认低轮次策略不变：默认 `safe` 仍为 12 轮；用户显式覆盖时最高允许 120 轮，支持小红书等复杂发布任务使用 `--max-rounds 80`。
- 修复 fast agent 显式长等待预算被硬压到 75 秒的问题；未显式配置时仍保留快路径短等待，避免普通任务变慢。
- Matrix 手机分发路径同步使用同一套显式预算逻辑，避免动态工作流与单机 quick-task 行为不一致。
- 手机任务运行时 stdout/stderr 现在会实时写入 Matrix 事件池；无论从工作台、CLI 还是 MCP/Codex 触发，超级矩阵工作台都能看到手机任务日志持续滚动。

## 发布与契约测试

- `phone:publish` 的契约测试从源码字符串扫描升级为真实 CLI 测试：启动假 APKClaw HTTP 服务并验证 `/api/lumi/agent/tasks` 请求体实际包含 `timeout_sec` 和 `max_rounds`。
- 更新 `loom-command-brain` 示例，将平台发布建议预算改为 `--max-rounds 80`。
