# LOOM 2.1.96 更新说明

## 智能体体验

- 审批操作现在立即返回，手机任务在后台继续执行，不再出现界面超时但手机仍在动作的割裂状态。
- 同一任务只显示一条连续的工具进度，成功自动收起，失败保留真实原因，不再重复刷“正在检查”或“正在调用”。
- 手机发布在登录受阻、网络异常或页面未完成时返回真实失败，不再把提示文字当作成功。
- 发布工具拿到终态后不再发起无意义的第二轮模型请求，减少等待和重复回复。

## 创作与手机相册

- 新增 Agnes Video V2.0 原生适配，修复视频请求误发到 DashScope 的问题。
- 兼容嵌套和字符串化的视频结果，生成完成后可在创作素材库正常预览。
- 图片和视频生成后自动传到选中手机的 `LOOM` 相册；传输前重新检查手机，不再受旧离线快照影响。
- 素材库准确区分文件字节数和生成分辨率，Range 播放与下载可用。

## 矩阵控制

- 人工接管期间自动续租，慢截图不会让点击、返回、主页或文字输入静默失效。
- 截图、UI 树和在线状态独立判断，UI 树暂时为空时仍可显示有效截图。
- 设备失败直接显示真实原因，不返回假成功。

## 品牌与账户

- 登录页、账户卡和 Agent 空状态统一使用打包内麓鸣 Logo，离线安装也能显示。
- 删除启动器内不可用的邮箱注册，只保留网页注册入口。
- 启动动画使用新的内置品牌视频，并保留静态封面降级。

## 安装与升级

- 修复直接覆盖安装时 `_up_` 内 Python/Node 持续运行、导致文件无法替换的问题。
- 安装器只停止当前安装目录内的进程，不会误关系统或其他项目的 Python/Node。
- 低页面文件导致 WMI/CIM 或 `taskkill` 不可用时，自动降级为直接进程终止。
- 安装器会先停止 LOOM 父进程树，再连续确认运行时文件已解锁；诊断日志写入 `%TEMP%\LOOM-installer-process-cleanup.log`。
- 在 32 位 NSIS 环境中通过 `Sysnative` 使用 64 位进程路径检测，无法确认后台退出时会中止安装并提示重试。
- 2.1.95 到 2.1.96 的真实占用覆盖测试已通过，升级数据保留。

## 安全边界

- 抖音任务继续强制草稿模式；不会擅自公开发布。
- 未登录或外部网络不可用时明确停止并提示，不伪造草稿成功。
- 日志只记录视频响应的结构和类型，不记录令牌或敏感响应值。
