# Luminode Desktop Agent Integration

## Positioning

Luminode is integrated as an optional desktop sidecar, not merged into the launcher process itself.

- The launcher discovers, configures, starts, stops, health-checks, and tails logs for the sidecar.
- Luminode handles desktop screenshots, clicks, typing, window detection, and chat-app RPA.
- OpenClaw calls it only through the launcher Bridge at `/api/desktop-agent/*`.
- OpenClaw must not read the desktop agent token directly.

## Private Configuration

The launcher owns this private config file:

```text
data/.openclaw/launcher/desktop-agent.json
```

Primary fields:

- `agentDir`: Luminode directory. Empty values should auto-discover a nearby `agents/luminode-desktop` folder.
- `port`: default local HTTP API port, currently `21900`.
- `token`: generated by the launcher and never displayed in plain text.
- `appType`: target app type such as `weixin`, `wework`, `dingtalk`, or `generic`.
- `autoStartHttpApi`: whether the launcher should start the sidecar HTTP API.

## Bridge API

The launcher Bridge exposes:

- `GET/POST /api/desktop-agent/status`
- `POST /api/desktop-agent/config`
- `POST /api/desktop-agent/start`
- `POST /api/desktop-agent/stop`
- `GET/POST /api/desktop-agent/health`
- `POST /api/desktop-agent/screenshot`
- `POST /api/desktop-agent/click`
- `POST /api/desktop-agent/type`
- `POST /api/desktop-agent/wechat/send`
- `POST /api/desktop-agent/wechat/unread`

## Security Contract

- Luminode should listen only on `127.0.0.1`.
- The local HTTP API must accept `X-Desktop-Agent-Token` or `Authorization: Bearer`.
- OpenClaw must call the desktop sidecar through the launcher Bridge.
- Auto-reply and real-contact message sending require clear user intent.
- Token values must not be written into public runtime context files.

## Packaging Plan

Longer-term portable packages should place the sidecar under:

```text
OpenClawFiles/agents/luminode-desktop
```

The launcher should then:

1. Find the packaged sidecar.
2. Verify its executable/runtime dependencies.
3. Start the local API on loopback.
4. Save private connection info under `data/.openclaw/launcher/desktop-agent.json`.
5. Expose a health summary in the desktop control page.

## Remaining Work
