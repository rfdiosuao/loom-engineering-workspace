# LOOM 2.1.110 更新说明

## 智能体手机目标

- 修复选择单台手机后，生成素材仍可能发送到全部已配置手机的问题。
- 智能体现在完整继承单台设备、设备组和“全部在线”目标范围。
- 模型不能自行添加会话范围之外的手机目标，避免误操作其他设备。

## 图片与发布链路

- 修复图片服务返回 HTTP 524 后拆分重复请求的问题，避免重复生成、迟到结果和潜在重复计费。
- 保留 400/422 参数兼容场景的安全单张降级能力。
- 发布工具遗漏标题时，可从用户明确标题或发布正文首行自动补全。
- 同一会话已有可用图片或视频时优先复用原素材，不再无意义重复生成。

## 智能体交互

- 参数修复过程不再生成重复失败卡片。
- 任务失败后自动清理“正在检查”“正在发布”等失效准备性回复。
- 已持久化的成功结果不会被后续失败事件误删。
- 后台日志新增任务类型与中文标签，便于区分手机、矩阵、媒体和组件任务。

## 运行时诊断

- 修复 Hermes 检测将文档标题分隔线误判为代码冲突标记的问题。
- 图片网关超时返回明确、可重试的中文错误，不再只显示笼统任务失败。
