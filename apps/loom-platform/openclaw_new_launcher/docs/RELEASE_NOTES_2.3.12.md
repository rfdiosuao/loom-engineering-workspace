# LOOM 2.3.12 更新说明

## 智能体执行可靠性

- 修复生图、生视频、素材传输和发布任务已提交后连接中断时，智能体错误判断为“未执行”并允许重复操作的问题。
- 修复后台任务状态查询中断时丢失任务真实状态的问题；现在会明确提示先核对结果，避免重复生成或重复发布。
- 修复模型跨轮复用同一个工具调用标识、但更换能力或参数时，后续操作被静默跳过的问题。
- 修复操作已经完成、仅界面事件同步失败时被错误标记为可重试的问题。

## 安全重试与错误提示

- 收紧 MCP、CLI 和麓鸣内置能力的执行前错误判定，不再把未知回执或无效结果误认为可安全重试。
- 已执行但结果待确认的任务会保留为不可自动重放状态，并提供中文核对提示。
- 精确保留缺少输入、目标、授权或环境配置等真正发生在执行前的可修复错误。

## 测试覆盖

- 增加后台任务提交、查询和取消响应中断的对抗测试。
- 增加工具调用标识冲突、完成事件同步失败及跨协议错误分类测试。
- 完成 Python、前端平台契约、Node 契约和生产构建回归。
