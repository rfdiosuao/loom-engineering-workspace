# LOOM 2.1.106 更新说明

## 商业授权调整

- 取消启动器全局授权墙，未激活商业授权也可以进入麓鸣并使用智能体、生图、生视频、模型账户和安装能力。
- 商业授权集中到手机连接与矩阵工作台；一份授权统一解锁设备检测、矩阵任务、实时画面、人工接管和手机素材传输。
- 手机连接页与矩阵工作台使用同一套授权状态，不再重复校验或出现页面之间状态不一致。

## Agent 与 CLI 授权边界

- 智能体调用矩阵状态、截图、任务下发、失败重试和手机发布时均校验手机矩阵授权。
- CLI/MCP 的全部手机命令统一归入 `matrix.devices`，包括只读状态与截图，避免通过外部 Agent 绕过商业授权。
- 桌面 RPA 和非手机能力不再被旧的全局商业授权判断误拦截。

## 安全恢复

- 授权到期或停用后仍保留矩阵取消、全部急停、手机事件停止和守护进程停止能力。
- 未授权页提供“紧急停止全部手机任务”入口，确保正在执行的手机任务始终可控。
- 安装升级继续在覆盖 `_up_` 运行时前定向停止当前安装目录内的 LOOM、Node 和 Python 进程，无法释放文件时会中止并写入诊断日志。

- 前端平台合同：129 项通过。
- Node 手机与媒体协议合同：62 项通过。
- TypeScript、Vite 生产构建与手机矩阵授权页响应式检查通过。
