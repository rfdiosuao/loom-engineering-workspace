# LOOM 2.3.4 更新说明

## 智能体工具路由

- 多台手机、指定设备组和单台手机任务现在分别进入矩阵或单机执行链，不再相互串用。
- 已有图片、视频的编辑、转换与相册传输会复用本地素材，只开放当前步骤需要的生成或传输工具。
- 生图、生视频、配置保存与接口测试按媒介类型精确分流，避免图片任务误调用视频能力。
- 发布内容、传入手机相册和普通手机操作相互隔离，降低误发布与重复执行风险。

## 能力可达性

- 修复手机快速任务、模板任务、读屏、主题查询和安装更新等正式中文名称无法稳定选中对应工具的问题。
- 修复矩阵进度、手机事件同步和媒体配置等动作被同域其他工具淹没的问题。
- 生产能力目录中的 MCP 工具均可通过自身名称与说明被智能体识别，历史能力 ID 保持兼容。

## 多步执行稳定性

- 素材查询后传输会继续绑定本次请求选中的设备，模型返回的越权设备 ID 不会覆盖用户目标。
- 多步任务只展示和执行当前阶段所需能力，减少重复检查、无关调用和失败刷屏。
- 新增生产能力目录、自然语言对抗路由和素材复用多步编排回归测试。

## 版本

- 产品版本统一升级到 `2.3.4`，安装包、前端、Rust 与发布元数据保持一致。
