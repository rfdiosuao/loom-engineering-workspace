# OpenClaw 启动器小白视角体感审查

记录日期：2026-06-16  
来源：子智能体“小白用户视角”审查 + 当前项目上下文  
范围：启动器首页、服务 / CLI、授权、统一设置、图像 / 视频、手机控制、桌面 RPA、Skills、环境检测、更新 / 在线包。

## 结论

当前启动器能力已经很完整，但默认体验仍偏“工程控制台”。普通用户第一次打开时，主要阻力不是功能缺失，而是：

1. 不知道第一步该做什么。
2. 配置项暴露太多，URL、Token、JSON、路径和 runtime 让用户紧张。
3. 长任务反馈不够持续，启动核心、桌面 RPA、视频生成、手机任务容易被误判为卡死。
4. 手机控制功能强，但信息密度太高，需要新手模式。

产品方向：保留高级能力，但默认给用户看“下一步、连接状态、常用任务、结果日志、修复按钮”。

## 问题清单

| 模块 | 电脑小白看到的现象 | 为什么影响体感 | 严重度 | 建议改法 |
| --- | --- | --- | --- | --- |
| 启动器首页 | 首页第一屏很漂亮，但标题“满船清梦压星河...”太有风格，新手不知道第一步到底点哪里 | 情绪价值有了，但“先启动服务 / 先授权 / 先配置模型”的任务优先级不够硬 | P1 | 首页保留标题，但右侧做成明确三步：启动核心、配置模型、连接手机或桌面 |
| 启动器首页 | “桌面 RPA”“手机控制台”“Skills”“图像 / 视频”同时出现 | 新手会觉得每个都要配置，心理负担大 | P1 | 首页增加“推荐下一步”卡片，根据状态只推一个按钮 |
| 启动器首页 | “接口网关、桥接来源、真实接口、星桥”等词混用 | 小白无法判断这是网络、模型、授权还是本机服务 | P1 | 用户侧统一叫“模型接口”“手机连接”“桌面组件”，工程词放到高级详情 |
| 服务 / CLI | 启动和停止只有 toast，启动慢时不知道是卡住还是正常等待 | 用户容易反复点按钮，造成更多异常 | P0 | 启动 / 停止期间做持续状态条：当前阶段、已耗时、预计上限、最近一行日志、失败后下一步 |
| 服务 / CLI | “420s 超时”“桥接运行信息”“Node 路径”“根路径”直接展示 | 小白看到路径和 PID 会紧张，以为自己要处理 | P2 | 默认显示“服务正常 / 异常原因 / 处理按钮”，路径类信息折叠到“高级排障” |
| 服务 / CLI | “检查更新”同时存在于服务页和统一设置页 | 用户不知道这是更新 OpenClaw 核心，还是更新启动器 | P1 | 拆名：服务页叫“检查核心更新”，设置页叫“检查启动器更新” |
| 授权内测 | 授权成功后自动启动服务很好，但失败时只看到原始错误 | 用户不知道是授权码错、网络错、还是服务器错 | P1 | 失败分类展示：授权码无效、服务器不可达、本机时间异常、已绑定其他设备 |
| 授权内测 | “发卡网站”“成员租约”“网关映射”偏运营 / 工程 | 内测用户容易误解，不知道跟自己有什么关系 | P2 | 用户态改成“获取授权”“当前账号”“可用额度”；工程态折叠 |
| 统一设置 | 模型、图像、视频、OpenAI OAuth、桌面 RPA 同步都在同一页 | 功能集中是好事，但新手会被多套 Key 和 URL 吓住 | P0 | 设置页做“普通模式 / 高级模式”。普通模式只放主模型接口和一键同步 |
| 统一设置 | OpenAI OAuth 代理字段出现，但用户不知道代理填什么 | unsupported_region 问题仍缺自解释 | P1 | 加“检测代理”按钮：显示能否访问 auth.openai.com，失败时给代理示例 |
| 统一设置 | “运行 openclaw onboard”像开发命令 | 小白不知道 onboard 是什么，点之前会犹豫 | P2 | 改成“打开 OpenClaw 配置向导”，小字再说明会运行 `openclaw onboard` |
| 统一设置 | 配置文件、环境变量、原始 JSON 仍在页面底部 | 虽然折叠了，但仍显得启动器“脏” | P2 | 挪到“环境检测 / 高级排障”，普通设置页只保留用户能理解的表单 |
| 图像 / 视频 | 切页状态保留已修，但缺少“任务中心”感 | 用户切回来能看到动画，但不知道历史任务是否还在后台跑 | P1 | 右上角增加“生成任务”小状态：排队中、生成中、成功、失败，跨模块常驻 |
| 图像 / 视频 | 缺少配置时提示“缺少图像网关 / 视频网关” | “网关”对小白不直观 | P2 | 改成“还没有配置图像接口”，按钮直接跳“去填写图像接口” |
| 图像 / 视频 | 视频生成耗时长，失败后多数是原始错误 | 用户不知道是超时、余额、模型名错、URL 错还是服务商慢 | P1 | 失败卡片分类：接口不可达、API Key 错、模型不存在、排队超时、无视频返回，并附复制诊断 |
| 手机控制 | 页面能力太多：设备、截图、任务、录屏、编队、任务库、计划、日志、诊断都在一页 | 最强模块也是最容易让新手迷路的模块 | P0 | 拆成 3 个清晰区：连接手机、执行任务、定时任务；高级诊断折叠到底部 |
| 手机控制 | “新增”设备后如果只是刷新页面 / 展开配置，新手会以为没反应 | 反馈不够强，用户不知道新增是否成功 | P1 | 点击新增后自动滚到配置区，配置区高亮 1 秒，标题改为“正在添加新手机” |
| 手机控制 | Token、Lumi 签名、重新配对、安全通道这些词很多 | 一旦失败，小白不知道该复制 Token 还是重新配对 | P0 | 做“一键修复连接”：依次检查 IP、Token、时间、Lumi 配对、APK 版本，最后只给一个结论 |
| 手机控制 | “同步模型到手机”失败时显示 Lumi signature 原始错误 | 用户无法判断是手机端还是电脑端问题 | P0 | 错误翻译层：`Invalid Lumi signature` 显示为“电脑和手机的安全配对失效，请点重新配对” |
| 手机控制 | 任务库里有 dry-run、unattended、risk level 等概念 | 对开发者清楚，对普通用户抽象 | P1 | 任务模式改成中文三档：只演练、执行但需确认、自动执行 |
| 手机控制 | 定时任务有“启动 / 停止 / 检查 / 创建计划 / 样例计划” | 新手不知道启动的是调度器，还是启动某个任务 | P1 | 顶部做定时任务总开关；计划卡片只保留启用、试运行、删除 |
| 手机控制 | 闲鱼自动化模板入口藏在任务库里 | 用户想做“闲鱼擦亮”时不一定知道要进任务库找模板 | P1 | 手机控制页顶部加“常用自动化”快捷卡：闲鱼擦亮、签到、读取发布状态、广告等待 |
| 桌面 RPA | “启动 / 停止”体感慢时只有 toast 和状态卡 | 用户点完不知道有没有真的启动，容易连点 | P0 | 启动按钮进入 loading 态并禁用，显示正在启动组件、等待健康检查、已就绪 |
| 桌面 RPA | “发送”按钮用于微信自动回复，但危险感不够明确 | 小白可能误触发真实微信发送 | P0 | 发送动作分两步：先写入草稿，开启允许自动发送后按钮才变成发送到微信 |
| 桌面 RPA | 组件未安装说明出现 manifest、sha256、OpenClawFiles 路径 | 新手不关心校验细节，只想知道能不能装 | P2 | 改成“首次使用需要下载桌面组件，约 XX MB”，校验细节放悬浮说明 |
| Skills | “安装 ZIP”对普通用户不友好 | 小白不知道 Skill 是插件还是模板，也不知道去哪下载 ZIP | P1 | 增加“从官方库安装”入口；ZIP 安装放次级按钮 |
| Skills | Skill 卡片仍显示 runtime、path、category | 用户会觉得这是开发者后台 | P2 | 默认只显示名称、用途、开关；路径和 runtime 放“详情” |
| 环境检测 | “环境检测要像报告，不像一堆报错弹窗”像内部宣言 | 用户更想知道“我现在能不能用” | P2 | 标题改为“检查启动器是否可以正常运行”，副标题给结果导向 |
| 环境检测 | 修复按钮不知道会改什么 | 小白会怕点坏环境 | P1 | 点修复前弹出将执行的动作清单，或显示“不会删除数据” |
| 更新 / 在线包 | 在线包首次启动会下载运行时层，但用户不知道下载多大、从哪下、失败怎么办 | 首装体感决定第一印象，下载黑盒会像卡死 | P0 | SetupGate 显示每个组件大小、当前下载源、速度、重试次数、切换镜像按钮 |
| 更新 / 在线包 | 自更新只说“下载完成后启动安装包并退出” | 用户会担心数据丢失 | P2 | 增加“不会删除 OpenClawFiles 和已有配置”的明确说明 |
| 全局反馈 | toast 4 秒自动消失，复杂错误来不及看 | 失败时用户需要复制给维护人员 | P1 | 错误 toast 增加“展开详情 / 复制诊断 / 打开日志” |
| 全局文案 | 有些地方很诗意，有些地方很工程化 | 风格不稳定：一会儿品牌感，一会儿开发控制台 | P2 | 标题短、动作明确、错误给下一步；诗意只留首页和少量空状态 |

## 优先级建议

### P0：先解决会让用户停下来的问题

1. 服务启动 / 停止的持续进度反馈。
2. 统一设置普通模式，隐藏多余工程配置。
3. 手机控制拆分为连接手机、执行任务、定时任务。
4. 手机连接的一键修复与错误翻译层。
5. 桌面 RPA 启停 loading 阶段反馈。
6. 桌面 RPA 发送动作二次确认或草稿优先。
7. 在线包首装下载进度和镜像切换。

### P1：提升交付体感

1. 首页推荐下一步。
2. 授权失败分类。
3. OpenAI 代理检测。
4. 图像 / 视频任务中心。
5. 视频生成失败分类和复制诊断。
6. 手机自动化任务模式中文化。
7. 常用自动化快捷卡。
8. 错误 toast 可展开、可复制、可打开日志。

### P2：降低工程感

1. 工程词统一替换到高级详情。
2. 路径、manifest、sha256、runtime 默认折叠。
3. Skills 用户态改名为“能力插件”。
4. 环境检测标题改成结果导向。
5. 文案风格统一。

## 产品方向

下一阶段不建议继续猛加功能。应先做一层“新手模式”：

```text
默认展示：连接状态、下一步按钮、常用任务、结果日志、修复按钮
高级折叠：路径、JSON、Token 明文输入、manifest、runtime、原始错误
```

新手模式不是删除高级能力，而是把高级能力放在用户真正需要排障时再出现。

## 解决记录（2026-06-16）

本轮已按上表逐条落地，`tsc` / `cargo check` / `vite build` 全部通过，并经两个审查子智能体交叉复查。

### 全局基础设施
- 错误提示可操作：danger 提示常驻不自动消失，新增「展开详情 / 复制诊断 / 打开日志」（`Shell.tsx`、`types.ts`、`appStore.ts`、`styles.css`）。
- 新增错误翻译层 `lib/errors.ts`（手机 / 授权 / 图像视频 / 通用四类）与 `lib/clipboard.ts`。
- 顶栏新增跨模块「生成中」常驻角标；侧栏工程词改口语化。

### 各模块（对应上表）
- 启动器首页：状态驱动「推荐下一步」单按钮 + 三步进度；接口网关→模型接口；保留诗意标题。
- 服务 / CLI：启停持续进度条（阶段 / 已耗时 / 上限 / 最近一行日志 / 失败重试）+ 防连点；路径 / PID / 超时折叠到「高级排障」；改名「检查核心更新」。
- 授权内测：失败四分类；发卡→获取授权、租约→当前账号、网关折叠、显示可用额度。
- 统一设置：普通 / 高级模式；OpenAI「检测代理」；onboard 按钮改「打开 OpenClaw 配置向导」（命令不变）；JSON / 环境变量 / 路径折叠；改名「检查启动器更新」。
- 图像 / 视频：失败分类卡片 + 复制诊断；「还没有配置图像 / 视频接口」+ 直达设置；生成状态行。
- 手机控制：拆三区 + 高级诊断折叠；一键修复连接（单一结论）；错误翻译（含 Lumi 签名）；新增设备滚动高亮；任务模式中文三档；定时任务总开关 + 计划卡精简；常用自动化快捷卡。
- 桌面 RPA：启停 loading 阶段 + 防连点；发送二次确认（默认写草稿、开关后才真正发送）；下载文案口语化 + 校验细节折叠。
- Skills：从官方库安装入口，ZIP 降级；卡片默认只名称 / 用途 / 开关。
- 环境检测：标题结果导向；修复前弹窗列动作 + 「不会删除你的数据」。
- 更新 / 在线包：总大小、实时速度、当前下载源 + 自动切换备用源（`bootstrap.rs` 新增 `dist://source` 事件）、「不会删除 OpenClawFiles 和已有配置」。

### 保留取舍
- 手动「切换镜像按钮」未做：下载器已自动按序切换备用源，界面现显示「源 N/M，已自动切换备用源」，手动切换意义不大。
- 首装总大小仅当 dist manifest 含 `bytes` 字段时提前显示（Rust 已支持读取），否则下载中按组件实际大小显示。
- 手机页少数二级路径仍走旧本地报错助手；主用户路径已全部接入翻译层。
