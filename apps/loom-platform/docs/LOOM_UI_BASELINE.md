# LOOM / 麓鸣 UI 基线

> 目的：以后任何 UI 改动先看这份文档。不要再在不同风格之间来回摇摆。

## 产品定位

LOOM，中文名“麓鸣”，是一套多智能体安装器和本地能力控制台。

它不是 OpenClaw 专用启动器，也不是营销官网。第一屏就应该让用户完成三件事：

1. 登录中转站并同步模型。
2. 安装或检查 Codex、Claude Code、opencode、OpenClaw、Hermes 等智能体。
3. 进入本地能力：桌面 RPA、手机 Agent、生图、生视频、CLI 自动化。

## 选定 UI 方向

固定采用“心流式浅色专业安装器”方向：

- 左侧窄黑色主导航，只放图标、短标题和登录入口。
- 主内容区使用浅暖灰背景，不使用星空大图、大片渐变、夸张 Hero。
- 内容以安装卡片、状态面板、详情面板为主。
- 页面标题只保留一句清楚标题，不堆解释文案。
- 弹窗用于登录、模型选择、确认高风险操作。
- 所有长说明进入详情、帮助、日志或诊断，不放在主 UI 上。

视觉目标：干净、稳定、像工具，不像宣传页。

## 信息架构

演示版主导航固定为 6 个入口：

| 入口 | 用途 |
| --- | --- |
| 启动器 | 总览、查看安装完成度 |
| 智能体 | 安装 Codex / Claude Code / opencode / OpenClaw / Hermes |
| 能力 | 手机 Agent、桌面 RPA、生图、生视频、CLI 自动化及任务状态 |
| 账号 | 中转站登录、访客模式、兼容授权码 |
| 模型 | 登录后选择默认文本、图片、视频模型 |
| 诊断 | 环境检查、日志、发布前自检 |

不要再恢复飞书、微信、广告视频、发布、Storyboard 等旧入口。相关能力如仍需要，只能放到“能力”里的暂未开放状态。

## 页面规则

### 启动器总览

只显示：

- 当前账号状态。
- 当前模型状态。
- 智能体安装状态。
- 核心服务状态。
- 一个主按钮：登录 / 安装智能体 / 打开能力。

不要放长段落。不要用大背景图。不要讲“超级强大”。

### 智能体安装页

采用左列表、右详情：

- 左侧：智能体列表 + 状态徽标。
- 右侧：名称、版本、大小、来源、安装状态、操作按钮。
- 主操作：检测 / 安装 / 启动 / 回滚。
- 高级信息默认折叠，只放 id、entry、installPath、URL。

状态必须覆盖：

- 未安装
- 准备中
- 下载中
- 校验中
- 安装中
- 配置中
- 检测中
- 已就绪
- 失败
- 可回滚

### 账号页

固定双入口：

- 中转站账号登录：主入口。
- 兼容授权码：折叠或次级入口。

登录后显示：

- 账号名
- 分组 / 订阅
- 可用额度或状态
- 已同步模型数量
- 退出登录

不要让用户手填 API Key 作为主流程。

### 模型页

模型选择必须登录后可用：

- 默认文本模型
- 默认图片模型
- 视频模型只展示和选择草案，不自动切 provider，直到后端任务链路稳定。

保存后写入启动器配置，用户不应该看到中转站 API Key。

### 能力页

只放真实能力入口：

- 桌面 RPA
- 手机 Agent
- 图片生成
- 视频生成
- CLI 自动化

图片和视频任务必须是异步任务。切到别的页面再回来，进度和结果不能丢。

## 文案规则

文案必须像产品帮助文案，不像聊天：

- 标题用名词：智能体、模型、账号、诊断。
- 按钮用动词：登录、同步模型、检测、安装、回滚、启动、停止。
- 错误用用户能理解的话：不要直接展示英文堆栈。
- 少解释，多给下一步。
- 不写空泛词：强大、领先、一键搞定一切、超级无敌。
- 不写口语化劝导：别急、先别、我们来。

示例：

| 场景 | 推荐文案 |
| --- | --- |
| 登录失败 | 登录失败，请检查账号、密码或网络代理。 |
| 模型为空 | 当前账号没有可用模型，请在中转站确认分组权限。 |
| 安装失败 | 安装失败，已保留原版本，可重试或回滚。 |
| 手机签名失败 | 手机安全配对失效，请重新保存设备并验证通道。 |

## 命名规则

- 主品牌：LOOM
- 中文名：麓鸣
- 窗口标题：LOOM - 麓鸣多智能体安装器
- 主程序：LOOM.exe
- 主目录：LOOMFiles
- 包名：LOOM-Portable-v{version}-{date}

允许作为兼容协议残留：

- `/api/lumi/*`
- `X-LUMI-*`
- `lumiLauncherId`
- `lumiLauncherSecret`
- `Invalid Lumi signature`

这些只能出现在协议层、日志或诊断，不作为主 UI 品牌展示。

## 设计禁区

- 不做星空背景。
- 不做大段 Hero。
- 不做卡片套卡片。
- 不把历史模块重新铺回侧边栏。
- 不在主 UI 显示本地路径、端口、token、installPath。
- 不在普通错误里显示 traceback、HTTP 原始报错、英文异常类名。

## 实施纪律

每次 UI 改动必须同步更新本文件或在交接文档写明原因。

每次提交前至少检查：

```powershell
rg -n "Lumi Agent|LumiClaw|OpenClaw / Lumi|澶|鏅|绋|鍚|閿" openclaw_new_launcher/src openclaw_new_launcher/data openclaw_new_launcher/python
npm run build
```
