# LOOM / 麓鸣白标与私有化交付商业计划书

日期：2026-07-05

本文档聚焦 LOOM / 麓鸣的第二条商业主线：不是单纯销售麓鸣标准客户端，而是把“中转站搭建 + 白标启动器 + 模型账号 + Agent 安装器 + 手机矩阵 + CLI/MCP 能力 + 行业模板”打包成一套可交付、可转售、可持续运营的 AI 自动化平台方案。

## 1. 执行摘要

LOOM / 麓鸣可以被包装为一套面向 AI 服务商、行业服务商、MCN、电商服务商、本地生活服务商和自动化工作室的白标 AI 自动化平台底座。

客户购买的不是一个普通安装器，而是一整套可以换成自己品牌、接入自己中转站、配置自己模型套餐、交付给终端用户的 AI 工具系统。

这套方案帮助客户完成四件事：

1. 搭建自己的模型中转站和账号体系。
2. 获得一套带自己品牌的 Windows 安装器和桌面工作台。
3. 将 Codex、Claude Code、opencode、OpenClaw、Hermes、手机 Agent、创作能力和 CLI/MCP 接入统一到一个入口。
4. 围绕模型额度、自动化模板、手机矩阵、企业服务和运维升级形成持续收入。

一句话定位：

> LOOM 白标方案帮助服务商把模型资源、Agent 工具和手机自动化能力，包装成一套属于自己品牌的 AI 自动化产品。

更商业化的表达：

> 不是卖 API，而是交付一套可换品牌、可开账号、可卖套餐、可部署到客户电脑上的 AI 自动化平台。

## 2. 市场背景

### 2.1 企业 AI 落地正在从“会聊天”进入“会干活”

过去一年，很多企业已经开始接触 ChatGPT、Claude、通义、DeepSeek 等大模型。但大多数企业的使用仍停留在“打开网页聊天”的阶段。

真正的企业需求不是让员工多一个聊天窗口，而是让 AI 参与实际工作：

- 安装和配置 AI 编程工具。
- 操作电脑软件。
- 管理手机设备。
- 发布内容。
- 处理素材。
- 执行固定流程。
- 记录任务结果。
- 沉淀可复用模板。

这意味着市场正在从“卖模型调用”转向“卖 AI 工作流交付”。

### 2.2 服务商缺少可复制的产品入口

很多 AI 工作室、自动化工作室、MCN 服务商、电商代运营公司都有客户资源，也知道 AI 自动化有价值，但缺少一套可交付产品：

- 会卖模型，但没有桌面客户端。
- 会搭 NewAPI，但没有品牌化软件入口。
- 会写脚本，但无法交给普通客户长期使用。
- 会做项目制交付，但每个客户都从零搭，成本高。
- 会演示 AI，但客户真正安装、配置、运行时容易失败。

他们需要的是一套“可以改成自己品牌、能发给客户安装、能登录自己的中转站、能持续更新”的产品底座。

### 2.3 终端企业不想自己折腾技术

终端企业老板通常不关心底层技术。他们关心的是：

- 能不能快速装上。
- 能不能用自己的账号登录。
- 能不能把模型额度卖给团队或客户。
- 能不能把重复流程交给 AI。
- 出问题有没有人负责。
- 能不能用自己的品牌对外展示。

LOOM 白标方案正好连接了这两类需求：上游服务商想卖 AI 自动化能力，终端企业想获得低门槛、可交付、可维护的 AI 工具。

## 3. 核心机会

### 3.1 从“工具销售”升级为“平台交付”

单卖一个客户端，客单价容易被压低；交付一整套白标平台，客单价和复购空间都会更高。

LOOM 的商业价值不只来自功能本身，而来自端到端交付链路：

```text
模型资源
  -> 中转站账号
  -> 白标安装器
  -> Agent 安装与配置
  -> 手机矩阵与任务模板
  -> CLI / MCP 给 AI 总控调用
  -> 任务记录与经验沉淀
  -> 持续运维和版本更新
```

这条链路一旦跑通，客户买到的不再是一个软件，而是一个可以继续销售、继续开账号、继续收服务费的 AI 产品体系。

### 3.2 白标客户本身就是渠道

白标客户可能是：

- AI 工作室。
- 自动化工作室。
- 短视频运营服务商。
- 电商代运营团队。
- 本地生活服务商。
- 企业软件服务商。
- 培训机构。
- 私域工具代理商。

这些客户本身有终端客户资源。他们购买白标方案后，可以用自己的品牌继续销售给下游客户。

因此 LOOM 不只是在卖软件，而是在搭建一套代理交付网络。

### 3.3 用标准底座承接非标准需求

企业自动化需求天然复杂，不同行业、不同平台、不同流程差异很大。如果每次都从零写项目，交付成本会越来越高。

LOOM 的策略是：

- 标准能力内核保持统一。
- 品牌、域名、模型、模块、文案、安装包走配置。
- 行业差异沉淀为模板和服务。
- 高风险动作保留确认和审计。

这样既能满足客户定制，又不会把主工程改乱。

## 4. 产品与交付内容

### 4.1 白标客户端

交付一套带客户品牌的 Windows 客户端。

可定制项：

- 产品中文名。
- 产品英文名。
- 应用图标。
- 安装器图标。
- 启动画面。
- 侧边栏 Logo。
- 主色和按钮颜色。
- 窗口标题。
- 默认下载页。
- 发布者信息。

客户端核心能力：

- 智能体安装。
- 模型账号登录。
- 模型同步。
- 自定义 API Key。
- 手机控制。
- 手机矩阵工作台。
- 生图 / 生视频。
- Agent 接入。
- CLI / MCP 能力开放。
- 安装、更新、回滚。

### 4.2 中转站搭建

为客户搭建或接入 NewAPI / OpenAI 兼容中转站。

交付内容：

- 服务器部署。
- 域名和 HTTPS。
- 上游模型渠道配置。
- 用户分组。
- 模型可见范围。
- 额度和倍率。
- 订阅或充值页。
- API Key 生成与管理。
- 模型同步接口。
- 基础运维说明。

目标是让客户拥有自己的账号体系和模型分发能力。

### 4.3 模型账号闭环

终端用户打开白标客户端后，不应该手动配置一堆 API Key。

理想流程：

1. 用户登录客户的中转站账号。
2. 客户端同步余额、套餐、模型列表。
3. 用户选择默认模型。
4. 客户端把可用模型配置给 Codex、Claude Code、opencode、OpenClaw、手机 Agent 和创作模块。
5. 退出登录时清理托管配置，保留自定义配置路径。

这条链路是白标产品的关键价值之一。

### 4.4 智能体安装器

白标客户端内置智能体安装页，用于管理第三方 Agent 和本地运行时。

支持对象：

- Codex。
- Claude Code。
- opencode。
- OpenClaw。
- Hermes。

交付能力：

- 前置环境检测。
- 安装状态检测。
- 一键安装。
- 启动。
- 更新。
- 回滚。
- 卸载。
- 日志导出。
- 模型配置。

这让白标客户不需要逐个教终端用户如何安装复杂工具。

### 4.5 手机矩阵工作台

手机矩阵是 LOOM 区别于普通 AI 工具的核心能力之一。

可交付能力：

- 手机设备连接。
- 多设备状态展示。
- 截图。
- 读屏。
- 执行简单任务。
- 执行模板任务。
- 任务队列。
- 任务日志。
- 失败原因。
- 设备分组。
- 后续扩展到多手机矩阵。

推荐产品口径：

> 手机矩阵用于把重复、规则明确、经过授权的手机端流程变成可监控、可重试、可沉淀的任务模板。

不应承诺：

- 绕过平台风控。
- 批量骚扰。
- 自动处理验证码。
- 自动完成支付。
- 保证平台不限制账号。

### 4.6 创作能力

白标产品可以内置基础创作能力：

- 文生图。
- 图生图。
- 生视频。
- 素材保存。
- 素材推送到手机。
- 与手机发布流程衔接。

这让白标客户可以包装成：

- 内容创作助手。
- 短视频运营助手。
- 电商素材生成助手。
- 本地生活素材助手。

### 4.7 Agent 接入

LOOM 的另一个关键能力是让 Codex、Claude Code 或其他 Agent 调用它。

交付内容：

- CLI。
- MCP Server。
- Agent 接入提示词。
- Skill 安装说明。
- 手机控制命令。
- 创作命令。
- 日志命令。
- 安全边界。

产品价值：

> 外部 Agent 不只是聊天，它可以通过 LOOM 调用本地工具、手机设备、创作模块和任务模板，成为真正的总控大脑。

## 6. 商业模式

### 6.1 一次性交付费

向白标客户收取首期交付费用。

交付范围包括：

- 中转站部署。
- 白标客户端。
- 品牌资源替换。
- 模型配置。
- 安装器构建。
- 基础验收。
- 基础培训。

适合用于覆盖初始实施成本。

### 6.2 年度授权费

白标客户每年支付授权费用，获得：

- 白标客户端使用权。
- 更新维护。
- 兼容性修复。
- 新版本同步。
- 基础技术支持。

这是最重要的稳定收入之一。

### 6.3 模型用量分成

如果模型由上游统一提供，可以按用量、额度或充值抽取一定分成。

可选模式：

- 按消耗比例分成。
- 按套餐差价分成。
- 按代理等级给折扣。
- 按月结算。

注意：

- 模型成本波动较大，必须保留调价机制。
- 不要承诺长期固定低价。

### 6.4 定制开发费

客户需要行业模板、特殊 UI、特殊流程或特殊部署时，按项目收费。

常见定制：

- 专属登录页。
- 专属订阅页。
- 专属官网。
- 行业任务模板。
- 手机流程模板。
- MCP 工具扩展。
- 数据报表。
- 企业权限。

### 6.5 运维服务费

按月或按年收取服务费。

服务内容：

- 服务器维护。
- 中转站更新。
- 模型渠道监控。
- 安装包更新。
- 客户问题处理。
- 远程排障。
- 备份和恢复。

### 6.6 培训和代理赋能费

为代理商提供：

- 销售话术。
- 演示脚本。
- 安装培训。
- 中转站后台培训。
- 客户交付 SOP。
- 常见问题库。

可单独收费，也可包含在代理版套餐中。

## 8. 交付流程

### 8.1 售前诊断

先确认客户属于哪类：

- 自用企业。
- AI 工作室。
- 代理商。
- 行业服务商。
- 技术团队。

再确认客户核心诉求：

- 只要中转站。
- 要白标客户端。
- 要手机自动化。
- 要行业模板。
- 要代理销售体系。
- 要私有化部署。

### 8.2 资料收集

客户需要提供：

- 品牌名。
- Logo。
- 图标。
- 主色。
- 域名。
- 服务器或部署要求。
- 模型渠道。
- 默认模型。
- 开放模块。
- 售后入口。

### 8.3 中转站部署

完成：

- 服务器配置。

- HTTPS。

- 上游模型接入。

至少验证：

### 8.6 交付培训

培训内容：

- 如何更新版本。

- 如何说明能力边界。

## 9. 护城河

### 9.1 端到端链路壁垒

普通竞争者可能只做其中一环：

- 只做中转站。
- 只做安装器。
- 只做手机自动化。
- 只做模型调用。
- 只做 Prompt 或模板。

LOOM 的壁垒在于把这些环节串成完整交付链路。

### 9.2 白标工程能力壁垒

白标不是换 Logo。真正难点在：

- 独立 manifest。
- 独立更新通道。
- 独立回滚包。
- 独立模型策略。
- 安装器签名和校验。
- 客户间配置隔离。
- 不把密钥打进包。

这些工程细节决定了白标能否长期维护。

### 9.3 手机矩阵和任务模板壁垒

手机自动化不是单次点击，而是：

- 设备管理。
- 状态监控。
- 任务分步。
- 失败重试。
- 截图读屏。
- 模板沉淀。
- 人工确认。

这部分越使用，经验越多，模板越稳，形成持续壁垒。

### 9.4 Agent 总控能力壁垒

LOOM 不是只给人点按钮，也给 Codex / Claude Code 调用。

CLI / MCP 让外部 Agent 可以成为总控大脑，调用：

- 安装器。
- 手机。
- 创作。
- 任务日志。
- 工作台。
- 模板。

这让 LOOM 从桌面工具升级为 AI 自动化基础设施。

### 9.5 交付经验壁垒

每个客户的部署、报错、模板、模型配置、手机兼容性都会形成经验。

经验一旦沉淀为 SOP、模板和诊断工具，新客户交付成本会越来越低。

## 10. 竞争分析

### 10.1 传统中转站

优点：

- 模型管理成熟。
- 用户和额度体系清楚。

不足：

- 只有网页和 API。
- 缺少桌面工具入口。
- 缺少 Agent 安装器。
- 缺少手机自动化。
- 缺少白标客户端交付。

LOOM 差异：

> 在中转站之上提供桌面客户端和自动化能力，让模型变成可交付产品。

### 10.2 自动化脚本工具

优点：

- 单点流程可定制。
- 技术团队能快速开发。

不足：

- 普通用户不会运行。
- 不好更新。
- 不好售后。
- 不好白标。
- 不好接账号和模型。

LOOM 差异：

> 把脚本能力包装成可安装、可登录、可更新、可回滚的软件产品。

### 10.3 单一 AI 客户端

优点：

- 使用简单。
- 交互好。

不足：

- 不属于客户自己的品牌。
- 很难私有化。
- 很难卖给下游。
- 缺少手机矩阵和本地自动化。

LOOM 差异：

> 面向服务商和企业交付，不只是终端聊天工具。

## 11. 风险与应对

### 11.1 模型上游不稳定

风险：

- 上游限流。
- 模型下线。
- 价格变化。
- Responses / Chat Completions 兼容问题。

应对：

- 多上游通道。
- 模型健康检查。
- 失败自动切换。
- 明确模型能力范围。
- 交付时不承诺永久固定模型。

### 11.2 第三方 Agent 更新破坏兼容

风险：

- Codex、Claude Code、opencode 配置格式变化。
- 启动参数变化。
- 登录策略变化。

应对：

- 配置写入前备份。
- 启动前检测。
- 回滚配置。
- 版本适配清单。
- 白标包定期更新。

### 11.3 手机自动化平台规则风险

风险：

- 平台风控。
- 账号限制。
- App UI 改版。
- 无障碍权限变化。

应对：

- 不承诺绕过规则。
- 外发动作人工确认。
- 使用模板授权。
- 任务日志审计。
- 将手机能力定位为合规自动化和流程辅助。

### 11.4 支付和财税合规风险

风险：

- 个人收款不合规。
- 订阅收费缺少主体。
- 发票和报税问题。

应对：

- 建议客户使用个体工商户或公司主体。
- 使用正规微信支付商户号。
- 订阅支付走服务端。
- 客户端不保存支付敏感信息。
- 税务问题交由客户和会计处理。

### 11.5 售后压力

风险：

- 白标客户能力不足。
- 下游客户问题复杂。
- 每个行业都想定制。

应对：

- 标准版边界清楚。
- 定制需求单独报价。
- 提供代理培训。
- 提供 FAQ 和诊断工具。
- 不无限免费改需求。
