# 启动器认证改造设计：登录制 + 模型/额度同步（三档并存）

状态：Phase 1 已落地（启动器直连 new-api 登录 + 本地模型同步）；Phase 2 代理登录/离线许可证待评审
日期：2026-06-22
适用：LumiClaw 启动器（引擎 OpenClaw）、发卡/授权服务器（license.heang.top）、中转站 new-api（api-cn.heang.top）

---

## 1. 背景与目标

当前启动器用「授权码」激活：发卡服务器签发 Ed25519 许可证，许可证里嵌入网关配置（baseUrl / apiKey / 模型）。痛点：用户要么贴授权码、要么手填 Key/URL/模型，且模型、额度无法实时联动。

目标：在**不砍掉离线能力**的前提下，新增「用中转站账号登录 → 自动同步可用模型与额度」的路径，降低上手门槛、便于招商分发与服务端统一管控。

核心原则：**登录是「获取授权的新方式」，不是替代离线许可。** 三档并存、互补、不打架。

---

## 2. 三档总览

| 档位 | 目标客户 | 认证方式 | 联网要求 | 模型来源 | 额度/管控 |
| --- | --- | --- | --- | --- | --- |
| **A. 登录档** | 大众 / SMB / 招商分发 | new-api 邮箱+密码登录（启动器内直接登录） | 登录时联网；之后离线宽限可用 | 登录后自动同步 new-api 可用模型，亦可手填覆盖 | new-api 实时额度硬控 |
| **B. 在线授权码档** | 现有付费用户 | 授权码 → 发卡服务器换签名许可证（现状） | 激活时一次 | 许可证内嵌网关 | 许可证有效期 + 服务端 |
| **C. 离线许可证档** | 气隙保密企业（自部署大模型） | 离线许可证文件导入，**绑 U盘** | **完全不用** | 内网自有模型（嵌入许可证或高级模式手填） | 许可证有效期（信任制） |

> A 和 B/C 的关系：**长期并存**。登录是新增主路径，授权码与离线许可证保留，分别服务弱网/线下/气隙场景。

## 3. 已确认的环境事实

- **中转站 = new-api**，公网主入口 `https://api-cn.heang.top` → 火山云 nginx 反代 → 赔钱云 NewAPI；`https://api.heang.top` 仅保留为兼容回退入口。
- 登录：**邮箱 + 密码**（`email_verification: true`），无 OAuth。**开放注册**（用户自助注册）。
- 额度：**自定义货币**（符号「陇」），开启 签到 / 充值 / 兑换码(redemption) / 订阅(subscription)。
- 已启用 `enable_drawing`（图像）、`enable_task`（视频/异步任务）。
- 后台具备 user / token / models / channel / subscription 管理。
- 发卡/授权服务器在火山云（`/opt/openclaw-license/`，Python http.server + sqlite + Ed25519 签名）。
- 一个账号允许**多设备**同时登录。

## 4. 档 A：登录 + 模型/额度同步（主路径，Phase 1）

当前 MVP 已实现：

- Bridge 新增 `/api/account/current`、`/api/account/login`、`/api/account/sync`、`/api/account/logout`。
- 启动器「授权内测」页新增「账号登录 / 授权码」双入口，旧授权码路径保留。
- 登录成功后写入本机 member session，并同步到 OpenClaw 主模型网关与图像配置。
- 「同步模型」会重新拉取中转站模型；如果中转站临时不可达，会保留上一次有效模型列表。
- 退出登录只清理 `managedBy = newapi_account` 的配置，避免误删用户手动配置。
- 视频模型目前只识别并展示，不自动切换视频 provider，避免误接到不兼容 task 接口。

### 4.1 登录流程（启动器内直接登录）

```
启动器「登录」表单（邮箱 / 密码）
  → POST https://api-cn.heang.top/api/user/login  {username(或email), password}
  → 拿到登录态（cookie/session 或 access_token）
  → 用登录态调：
       GET /api/user/self        # 账号信息、额度(quota/used_quota)、分组(group)、套餐
       GET /api/token/           # 该用户的令牌(sk-)列表；无则 POST /api/token/ 新建一个
       GET /api/user/models 或 /v1/models（带 sk-）  # 可用模型清单
  → 启动器本地保存「短期凭证 + 令牌 + 模型缓存 + 额度快照」
  → 用令牌(sk-)作为各接口的 API Key，base = https://api-cn.heang.top/v1
```

> new-api 接口名以**实际部署版本为准**，落地前在 `api-cn.heang.top` 上逐个核对（见 §8 待核对接口）。

### 4.2 模型同步映射（→ 启动器现有三套配置）

启动器现有：聊天网关（`data/.openclaw/agents/main/agent/auth-profiles.json`）、图像（`imgapi_config.json`）、视频（`video_config.json`）。

- 同步逻辑：登录后用同一 `base + sk-`，把 new-api 返回的模型按用途归类填入三套配置：
  - 文本/对话模型 → 主模型网关（base/apiKey/model）。
  - 图像模型（drawing / `/v1/images/generations`）→ `imgapi_config.json`。
  - 视频模型（task）→ `video_config.json`。
- **自动同步 + 可手填覆盖**：默认自动填；用户在「高级模式」可手动改/锁定某项（覆盖同步值）。
- 默认模型：取账号分组默认或清单第一个；用户可在下拉里改。
- 待确认：**视频**在 new-api 是否 OpenAI 兼容、同 `/v1` 出（启动器现用 agnes-video provider）；若是另一套 task 流程，需要一层映射适配（见 §8）。

### 4.3 客户端凭证存储与离线宽限

- **只存短期凭证**：保存「令牌(sk-) + 模型缓存 + 额度快照 + 登录态有效期」，**绝不存明文密码**。Windows 用 DPAPI / 凭据管理器加密存储。
- **离线宽限**：登录一次后缓存上述内容，**离线可用 14 天**（可配 7–30）；到期需联网重新登录。
- **额度是硬控制**：调用额度由 new-api 实时扣减/拦截，额度用完即停，与宽限期无关 → 宽限期可放宽松，靠额度做硬约束。
- **退出登录**：清理本地令牌、模型缓存、账号信息、登录态。

### 4.4 客户端 UI 改动（启动器）

- 「授权内测」页（`LicensePage.tsx`）增加 **登录 Tab**：邮箱/密码、登录按钮、注册入口（跳 `api-cn.heang.top` 注册）、登录后展示：账号 / 套餐 / 剩余额度 / 默认模型 / 模型调用失败原因。
- 登录成功后**自动启动核心服务**（与授权码激活后的行为一致）。
- 「统一设置」普通模式：登录态下隐藏手填 Key，显示「来自登录账号的模型」；高级模式可手填覆盖。

### 4.5 安全

- 仅 HTTPS；登录接口**防爆破限流**（new-api 侧 + 客户端退避）。
- 令牌等同 API Key，按密钥规格掩码/加密。
- Cloudflare 风险：桌面端直 POST 登录可能偶发 CF 人机校验 → 见 Phase 2 代理加固。

## 5. 档 B：在线授权码（保留现状）

- 不改动现有授权码 → 发卡服务器换 Ed25519 签名许可证 → 内嵌网关的流程。
- 与登录档并存：授权码继续服务「不想注册账号 / 一次性付费 / 线下交付」的用户。
- 可选打通：new-api 的**兑换码(redemption)**可作为「充额度」入口，与授权码区分用途（授权码 = 开通启动器；兑换码 = 充中转站额度）。

## 6. 档 C：离线许可证文件（气隙企业，自部署模型）

面向「只有内网、用自部署大模型、连发卡服务器都到不了」的保密企业。**完全零联网。**

### 6.1 许可证文件

- 发卡方用现有 Ed25519 私钥**离线签发**一个许可证文件（JSON + 签名），通过 U盘/文件线下交付。
- 内容：组织名、到期时间、功能位、**绑定标识**、**内网网关配置（可选）**。
- 启动器「导入离线许可证」入口：选文件 → 验签（内置公钥）→ 激活，**全程不联网**。

### 6.2 绑定方式：绑 U盘

- 绑定到 **U盘（USB 卷序列号 / 硬件 ID）**：许可证只在该 U盘上有效，随 U盘走（契合便携 U盘产品形态）。
- 校验：启动器读取所在卷/设备序列号，与许可证中的绑定值比对。
- 备选：也可绑机器码（企业固定机器）。**默认绑 U盘。**

### 6.3 自部署模型网关

- **嵌入许可证**：发卡方签发时把企业内网 OpenAI 兼容端点（URL + key + 模型名）写进许可证，开箱即用；或
- **企业 IT 高级模式手填**：企业自己在「高级模式」配内网地址（更灵活，企业 IT 完全接受）。
- 两者都支持，默认走「嵌入」。

## 7. 发卡服务器角色与 Phase 2 加固（代理模式）

- **Phase 1**：启动器直连 new-api 登录（最简单，先上）。new-api base 做成**可配置**（默认 `api-cn.heang.top`）。
- **Phase 2（加固，可选）**：把 **license.heang.top 做成「登录代理」**：
  - 启动器只连发卡服务器；发卡服务器在后端用 **new-api admin token** 验证账号、取令牌、查额度，再用 Ed25519 **签一张短期会话**返回。
  - 好处：避开 Cloudflare 对客户端登录的人机校验；中转站细节与 provider key 全藏服务端；以后换中转站软件不打挂客户端；离线宽限 = 签名会话有效期。
  - 前置条件：需要一个 **new-api admin token / 专用服务账号**配给发卡服务器（待你提供）。

## 8. 待核对的 new-api 接口（落地前在实例上逐个验证）

| 用途 | 预期接口（以部署版本为准） | 备注 |
| --- | --- | --- |
| 登录 | `POST /api/user/login` | 返回登录态（cookie/access_token），核对字段名 |
| 账号/额度 | `GET /api/user/self` | quota / used_quota / group / 套餐 |
| 令牌列表/新建 | `GET /api/token/` · `POST /api/token/` | 拿/建 sk- 令牌；鉴权方式核对 |
| 可用模型 | `GET /api/user/models` 或 `/v1/models`(带 sk-) | 模型清单 + 分组限制 |
| 文本 | `POST /v1/chat/completions` | OpenAI 兼容 |
| 图像 | `POST /v1/images/generations`（drawing） | 是否同 base/令牌 |
| 视频 | task 接口（`enable_task`）或 `/v1/video/...` | **重点核对**：是否 OpenAI 兼容、能否被启动器现有视频客户端直接用 |

## 12. 风险

- 弱网/Cloudflare：客户端直登可能偶发被拦 → 代理模式或重试退避缓解。
- 中转站单点：api-cn.heang.top 不可用时只读和幂等登录请求可回退 api.heang.top；验证码发送与注册等有副作用请求禁止自动重放。
- 账号共享：多设备允许 → 靠 new-api 额度与风控，不靠设备数硬限。
- 凭证安全：只存短期令牌、加密存储、退出即清。
