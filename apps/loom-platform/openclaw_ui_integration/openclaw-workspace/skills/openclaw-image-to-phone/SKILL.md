# OpenClaw Image To Phone

Use this skill to move generated or existing PC images into the connected Android phone gallery through the phone Agent.

## When To Use

Use this workflow when:

- an image was generated by OpenClaw or the launcher;
- the user asks for a generated picture on the phone;
- a local image needs to appear in Android Gallery;
- the user asks for app icons, product images, posters, QR images, or design assets and the phone is connected;
- `runtime-context.json` says phone image import is configured or available.

## Default Behavior

After generating an image, send it to the phone automatically when all conditions are true:

1. The image is generated by the current task, not an arbitrary personal file.
2. Phone Agent config is available.
3. Token is available through runtime config, environment variables, launcher state, or explicit user input.
4. The user has not disabled automatic phone sync.

If any condition is false, save the image locally and provide a retry command.

## Procedure

1. Read `runtime-context.json`.
2. Locate the image file. Prefer an existing saved file under `data/generated-images/`.
3. If only a base64/data URL exists, save it as PNG under `data/generated-images/` before upload.
4. Use `capabilities.phoneAgent.defaultAlbum` from `runtime-context.json`; fallback to `OpenClaw`.
5. Upload with:

```powershell
npm run phone:image -- --image <image-path> --album <album>
```

6. For image-to-image editing, use:

```powershell
npm run phone:image:edit -- --reference-image <image-path> --prompt "<edit instruction>" --album <album>
```

7. Report the local path and phone path.

## Output Format

Return a short completion note:

```text
图片已生成并发送到手机相册。
本地：<local-path>
手机：Pictures/<album>/<filename>
```

If upload fails:

```text
图片已保存在本地，但手机上传失败：<reason>
本地：<local-path>
重试：npm run phone:image -- --image <local-path>
```

## Safety

- Do not print the phone token.
- Do not upload unrelated personal files without asking.
- Do not delete the local image after upload.
- Do not claim success unless the upload response succeeds.
