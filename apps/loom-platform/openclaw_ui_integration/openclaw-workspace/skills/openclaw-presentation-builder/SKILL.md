# OpenClaw Presentation Builder

Use this skill when the user asks for a PPT, slide deck, presentation outline, roadshow deck, business report deck, competitor analysis deck, or HTML presentation.

## Workflow

1. Identify audience, goal, and decision the deck must support.
2. Build the story before designing slides:
   - title;
   - context;
   - problem;
   - insight;
   - plan;
   - evidence;
   - next action.
3. Keep one main idea per slide.
4. Prefer evidence-based charts, screenshots, tables, and concrete examples over decorative filler.
5. If producing a web PPT, create a single self-contained HTML file unless the user asks for a PPTX.

## Deck Skeleton

```markdown
# Deck Title

## Slide 1 - Title
Purpose:
Visual:
Speaker note:

## Slide 2 - Executive Summary
...

## Slide 3 - Current Situation
...

## Slide 4 - Key Findings
...

## Slide 5 - Recommendation
...

## Slide 6 - Next Steps
...
```

## Style Rules

- Keep slide text short.
- Use strong section titles.
- Put numbers and conclusions up front.
- For Chinese business decks, use direct, polished Chinese and avoid empty slogans.
