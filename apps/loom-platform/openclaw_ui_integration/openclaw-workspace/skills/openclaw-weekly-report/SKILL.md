# OpenClaw Daily And Weekly Reports

Use this skill when the user asks for a daily report, weekly report, progress summary, project status update, work log, or boss-facing update.

## Workflow

1. Collect evidence from the user's notes, task logs, changed files, chat snippets, or explicit bullet points.
2. Group work into outcomes, not chronological trivia.
3. Separate:
   - completed work;
   - in-progress work;
   - blockers and risks;
   - next steps;
   - metrics or deliverables.
4. Make the tone fit the audience:
   - manager: concise, outcome-focused, no implementation noise;
   - team: include blockers, dependencies, and next actions;
   - personal review: include learnings and priorities.
5. Do not exaggerate progress. If evidence is weak, label the item as "planned" or "pending verification".

## Style

Use polished Chinese for Chinese office reports. Prefer short, concrete bullets and quantified outputs when available.
