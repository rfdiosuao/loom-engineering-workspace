# Lumi 私人版 UI 设计方案

> 目标：为“自己使用的 Lumi 版本”定义一套区别于客户交付版的高质感视觉方向。  
> 定位：个人 AI 创作工作站、OpenClaw 控制台、广告视频生产指挥舱。  
> 原则：品牌感强、科技感强、专业但不花哨，优先保证长期可维护。

## 1. 版本定位

Lumi 私人版不是客户白标版，也不是普通启动器。它应该像一个“AI 工作站”：

- 可以启动和管理 OpenClaw 核心服务。
- 可以配置模型、API、飞书/微信机器人和 Skills。
- 可以做 AI 生图、AI 视频和广告视频工作流。
- 可以作为你自己的演示版、内部测试版、产品原型版长期使用。

这版 UI 可以比客户版更有个性，因为它不需要照顾所有客户审美。它的任务是让人一眼觉得：这是一个认真做出来的 AI 创作平台。

## 2. 品牌命名

主品牌名：

```text
Lumi
```

建议副标题：

```text
AI Creative Console
```

中文副标题可选：

```text
智能创作工作站
```

窗口标题建议：

```text
Lumi - AI Creative Console
```

左上角品牌区建议显示：

```text
Lumi
AI Creative Console
```

不要再出现客户版的“永浩科技 - 智能AI服务平台”，避免私人版和交付版混淆。

## 3. 视觉方向

推荐方向：**深色高级科技感 + 香槟金品牌光 + 控制台质感**。

关键词：

- 深色
- 金属感
- 专业软件
- AI 控制台
- 创作工作站
- 低调但有锋芒

不建议做成：

- 大面积紫色渐变
- 普通后台管理系统
- 太像客户版白色商务 UI
- 花里胡哨的营销落地页
- 到处发光导致看不清内容

Lumi 应该像“每天真会用的专业工具”，而不是一次性宣传页。

## 4. Logo 使用规范

本版使用：

```text
D:/Axiangmu/AUSTART/logo.ico
```

需要转出并统一使用：

- `icon.ico`：Windows exe / 任务栏图标
- `32x32.png`：窗口小图标
- `128x128.png`：侧栏品牌区
- `logo.png`：主题 logo
- `logo-square.png`：紧凑导航和状态卡使用

使用位置：

- Windows 任务栏
- exe 文件图标
- 窗口左上角
- 侧边栏品牌卡
- 授权页
- 关于/帮助页
- 打包后默认主题目录

验收要求：

- 任务栏不是旧 OpenClaw 图标。
- 文件夹里的 `OpenClaw.exe` 显示新 logo。
- 应用内左上角 logo 不变形、不丢失、不出现破图。
- 深色和浅色模式下 logo 都清晰。

- 5% 青色/红色/绿色状态色

注意：金色只做“品牌和重点”，不要把整个页面都刷成金色。

## 6. 字体与排版

推荐气质：

- 标题：更硬朗、更像仪表盘。
- 正文：清晰、耐看、长时间使用不累。
- 日志：等宽字体，保持专业控制台气质。

建议：

```text
品牌名：字重大、字距稍微放松
页面标题：24-28px
模块标题：15-18px
表单标签：12-13px
按钮文字：13-14px
日志字体：Consolas / JetBrains Mono / Cascadia Mono
```

不要使用过大的 hero 字体。Lumi 是工具，不是官网。

## 7. 总体布局

推荐布局：

```text
┌──────────────────────────────────────────────┐
│ 顶部标题栏：Logo / Lumi / 状态 / 快捷操作      │
├──────────────┬───────────────────────────────┤
│ 左侧导航      │ 主工作区                         │
│ 品牌卡        │ 当前模块标题                       │
│ 功能入口      │ 功能内容 / 表单 / 控制台 / 工作台    │
│ 状态卡        │                                 │
└──────────────┴───────────────────────────────┘
```

左侧导航建议：

- 宽度 260-286px。
- 使用紧凑品牌卡。
- 图标使用方形或圆角小徽章。
- 当前选中项用金色边线或青色光条。
- 底部放授权/API/服务状态。

主工作区建议：

- 不要再像“一个 UI 套在另一个 UI 里面”。
- 主区背景、标题栏、侧栏需要统一成一套视觉语言。
- 页面内容用清晰分区，不要卡片套卡片。
- 日志、视频预览、广告工作台这种核心区域要有足够空间。

## 8. 首页仪表盘

私人版建议增加一个默认首页，叫：

```text
Lumi Console
```

首页展示：

- OpenClaw 状态：运行中 / 未启动
- Bridge 状态：FastAPI / Legacy
- 授权状态：已授权 / 未授权
- API 状态：已配置 / 未配置
- 飞书机器人：已安装 / 未绑定
- 微信机器人：已安装 / 未绑定
- Skills：已启用数量
- 快捷入口：启动服务、广告视频、AI 生图、AI 视频、Skills

这个首页会让 Lumi 看起来像完整产品，而不是只是一堆菜单。

## 9. 关键页面设计

### 9.1 服务日志

方向：专业终端。

要求：

- 深色日志区。
- 支持滚轮上下滚动。
- 默认不强制跟随底部。
- “跳到底部”按钮明确有效。
- 有导出日志、清空日志按钮。
- 日志文字对比度足够高。

### 9.2 授权码

方向：安全控制面板。

要求：

- 输入框必须明显。
- 授权状态要一眼看懂。
- 显示安装 ID / 设备绑定说明，但不要吓人。
- 私人版可以展示更多调试状态，客户版隐藏。

### 9.3 API 配置

方向：模型接入中心。

要求：

- `Url 链接` 文案统一，不再叫“中转站地址”。
- 模型要支持选择和自定义输入。
- 已配置状态要明确。
- 私人版可以展示当前 provider、model、baseUrl。

### 9.4 AI 生图

方向：创作面板。

要求：

- Prompt 输入区要大。
- 上传参考图入口明显。
- 生成结果用画廊布局。
- 支持打开输出目录。
- 结果图可预览、保存。

### 9.5 AI 视频

方向：视频生成控制台。

要求：

- 支持 DashScope / 可灵 / Seedance 等多模型方向预留。
- 视频生成后必须能预览。
- 下载后提示保存路径。
- 文生视频 / 图生视频切换清楚。
- API Key 默认不保留，用户每次自行填写或由配置中心决定。

### 9.6 广告视频工作台

方向：创作者分镜台。

要求：

- 产品图上传入口明显。
- 三视图、首尾帧、九宫格要形成流程，而不是散按钮。
- 每个镜头可编辑提示词。
- 九宫格应该能服务于多个镜头，不要让用户猜。
- 输出结果要能给专业人员验收。

### 9.7 飞书 / 微信机器人

方向：插件绑定终端。

要求：

- 安装日志弹窗可滚动。
- 二维码输出区域足够高，不能截断。
- 微信只支持扫码，不展示 ID / Key 绑定。
- 飞书支持扫码或 App ID / Secret 手动配置。
- 插件检测失败时要告诉用户“未检测到 / 已安装但未绑定 / 命令执行中”。

### 9.8 Skills

方向：扩展中心。

要求：

- 支持上传 Skill zip。
- 支持启用 / 禁用 / 卸载。
- 支持打开 SkillHub。
- 显示来源：上传安装 / OpenClaw 扩展目录 / node_modules。
- 不执行未知脚本，保持安全边界。

## 10. 交互动效

动效应该少但精：

- 应用启动时：轻微淡入，不要闪白。
- 页面切换：100-160ms 淡入或位移。
- 按钮 hover：边框微亮，不要大幅跳动。
- Toast：3 秒自动消失。
- 弹窗：背景轻微模糊，但不要模糊到看不清日志。
- 进度状态：用细线进度或状态点，不要大面积 loading。

禁止：

- 大面积旋转动画。
- 按钮乱跳。
- 每个卡片都发光。
- 切换主题时出现白屏或浅深闪烁。

## 11. 私人版与客户版差异

| 项目 | Lumi 私人版 | 客户交付版 |
| --- | --- | --- |
| 品牌 | Lumi | 客户品牌 |
| 风格 | 更强科技感，更有个性 | 稳重、清晰、少冒险 |
| 调试信息 | 可保留较多 | 尽量隐藏 |
| 环境诊断 | 可以保留 | 交付时默认隐藏 |
| 插件日志 | 可详细 | 简化提示 |
| 主题切换 | 可开放 | 视客户需求 |
| 首页仪表盘 | 建议保留 | 可按客户定制 |

私人版可以是“展示能力”的版本，客户版是“稳定交付”的版本。

## 13. 实施顺序

建议分 4 步，不要一口气全改：

1. 品牌替换
   - 应用名改为 Lumi。
   - 替换 exe / 任务栏 / 应用内 logo。
   - 替换窗口标题。

2. 主题升级
   - 加 Lumi 深色主题。
   - 修复启动瞬间浅色闪烁。
   - 调整侧栏、标题栏、按钮、弹窗、日志区。

3. 首页仪表盘
   - 新增 Lumi Console 首页。
   - 做服务/API/授权/插件/Skills 状态卡。
   - 提供核心快捷入口。

4. 创作工作台强化
   - 优化 AI 生图、AI 视频、广告视频工作台。
   - 修复预览、路径提示、上传入口、镜头提示词编辑。

## 14. 验收标准

Lumi 私人版完成后，应满足：

- 打开应用第一眼就是 Lumi，不是客户版或 OpenClaw 默认壳。
- exe 图标、任务栏图标、窗口 logo、应用内 logo 全一致。
- 深色主题启动不闪白。
- 左侧导航和主工作区没有明显色差或拼接感。
- 服务日志可滚动、可导出、可清空。
- AI 视频生成后能预览，并能提示保存路径。
- 飞书/微信二维码不截断。
- API 配置、授权、Skills、广告视频入口清晰。
- 打包后 `verify-release.ps1` 通过。

## 15. 当前建议

下一步建议先做“品牌替换 + Lumi 深色主题”，这是投入最小、效果最明显的一步。

优先改：

- `logo.ico` 转图标资源。
- Tauri `productName` / 窗口标题。
- 应用内品牌名和副标题。
- 默认主题改为 Lumi。
- 侧栏和顶部标题栏统一深色金属风。

等这一版视觉气质立住，再继续做首页仪表盘和广告视频工作台强化。
