# OpenClaw 售后排障手册

## 1. 启动失败

### 现象

- 点击启动后提示 Bridge 启动超时
- 页面卡在“启动中”
- 服务日志没有新内容

### 排查

1. 检查是否有旧的 OpenClaw 实例占用端口。
2. 打开环境诊断，执行一键修复。
3. 检查本地 Node 目录是否完整。
4. 查看服务日志里是否有 Python / Node 报错。

### 处理

- 结束占用进程后重新启动
- 重新打包或重新解压完整文件
- 如仍失败，导出日志发送给售后

## 2. 微信插件没有二维码

### 现象

- 点击安装后没有二维码
- 显示“未检测到插件”
- 右侧日志停在安装中

### 排查

1. 确认点击的是“安装微信机器人”。
2. 看日志里是否有安装命令输出。
3. 确认网络可访问微信绑定服务。
4. 确认二维码区域没有被窗口裁切。

### 处理

- 放大窗口后再看二维码区域
- 复制日志中的链接到浏览器打开
- 若仍异常，重装插件后重新扫码

## 3. 飞书插件无法安装

### 现象

- 点击安装后一直卡住
- 右侧日志显示命令启动但无结果
- 二维码不完整或无法识别

### 排查

1. 检查命令行输出是否完整。
2. 检查 App ID / App Secret 是否填写。
3. 若二维码显示不全，尝试滚动查看完整内容。
4. 确认飞书开放平台应用配置正确。

### 处理

- 重新点击安装
- 手动补齐 App ID / App Secret
- 复制登录链接到浏览器完成绑定

## 4. AI 视频生成后无法预览

### 现象

- 任务显示成功，但下方没有播放器
- 播放器黑屏
- 下载按钮无提示

### 排查

1. 检查任务是否真的返回了视频地址。
2. 确认视频文件没有被系统拦截。
3. 检查下载路径提示是否有显示。

### 处理

- 刷新页面重试
- 重新提交任务
- 使用导出日志核查后端返回内容

## 5. 日志无法滚动

### 现象

- 日志固定在底部
- 不能用鼠标滚轮查看历史内容
- 跟随底部按钮没有反应

### 处理

- 确认当前版本已更新到带滚动日志的版本
- 重新启动后再次查看
- 若仍异常，把日志截图和导出文件发给售后

## 6. 授权异常

### 现象

- 提示未授权
- 激活成功后状态没更新
- 授权码输入后 Bridge 启动异常

### 排查

1. 检查授权码格式是否正确。
2. 刷新授权状态。
3. 检查网络是否可访问授权服务。
4. 查看日志是否有激活失败详情。

### 处理

- 重新输入授权码
- 刷新状态后再启动
- 必要时重新打包最新版本

## 7. 建议客户提交的信息

- 版本号
- 系统版本
- 问题截图
- 导出的日志文件
- 出错前后的操作步骤
