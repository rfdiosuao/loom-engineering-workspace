# OpenClaw 授权后台保姆级操作手册

> 适用版本：当前 `license_server` 会员/授权后台  
> 目标：让销售、售后、运营直接按步骤操作月卡、套餐、网关和历史授权码，不依赖开发者口头解释。

## 1. 这个后台能做什么

这个后台负责四件事：

1. 生成授权码。
2. 把普通授权码改成月卡/会员套餐。
3. 批量修改已发出的授权码。
4. 管理会员网关配置，让客户端刷新后拿到最新权限。

它不是计费系统，不负责自动扣量和自动续费。  
它更像“授权中心 + 套餐配置中心”。

## 2. 登录方式

后台地址：

`https://license.heang.top/admin`

登录需要管理员 Token。

Token 来源：

- 服务器上的 `admin_token.txt`
- 服务端环境变量 `LICENSE_ADMIN_TOKEN`

建议：

- Token 只保留在内部人员手里。
- 不要直接发给客户。
- 如果怀疑泄露，立刻更换并重启服务。

## 3. 推荐的生产配置模型

### 3.1 月卡套餐的标准字段

建议每个月卡至少带这些字段：

- `客户名称`
- `版本`
- `到期日期`
- `功能权限`
- `会员模式`
- `套餐`
- `网关 Base URL`
- `通用 Token / API Key`
- `图像 API Key`
- `视频 API Key`
- `默认模型`
- `可用模型`
- `配额 JSON`

### 3.2 推荐的套餐命名

建议统一：

- `monthly`：月卡
- `quarterly`：季卡
- `yearly`：年卡
- `vip_monthly`：高配月卡

不要每次都临时手写一套新名字，否则后面售后很难查。

### 3.3 推荐的配额写法

示例：

```json
{
  "image": 100,
  "video": 20,
  "phoneAgent": true,
  "desktopAgent": true
}
```

如果你暂时只关心图像和视频，就先保留：

```json
{
  "image": 100,
  "video": 20
}
```

## 4. 新建月卡的标准流程

### 第零步：选择套餐模板

后台顶部的“套餐模板”区域会显示当前可用模板。常用模板包括：

- `monthly`：月卡
- `quarterly`：季卡
- `yearly`：年卡
- `vip_monthly`：高配月卡

在生成授权表单里选择套餐模板后，后台会自动填入有效期、功能、套餐名、网关地址、默认模型、可用模型和配额。网关 Token 不会在页面回显，但模板里已配置时，服务端会在生成授权码时自动使用。

### 第一步：填客户信息

左侧表单里填写：

- 客户名称
- 版本
- 到期日期
- 每码激活次数
- 功能权限

### 第二步：打开会员模式

把 `会员模式` 选成：

- `会员 / 托管网关`

这一步表示这个授权码不只是普通授权，而是会同时携带网关配置。

### 第三步：填网关配置

把下面几个字段填完整：

- `网关 Base URL`
- `通用 Token / API Key`
- `图像 API Key`
- `视频 API Key`
- `默认模型`
- `可用模型`
- `配额 JSON`

图像和视频 API Key 可以单独配置。留空时客户端会继承通用 Token；如果你有独立的生图或视频供应商，就分别填写专用 key。
客户端读取会员配置时会优先使用 `gatewayProfile` / 会员会话里的专用 key，空值再回退通用 Token。

如果这些空着，客户端拿到的就是普通授权，不会有会员网关能力。

### 第四步：生成授权码

点 `生成授权码`。

生成后要立刻做两件事：

- 复制保存到售后文本
- 导出 TXT 给销售/客户

不要把“完整授权码”留在后台当记忆体。

## 5. 把已经生成的普通码改成月卡

这是最常见的操作。

### 单个修改

1. 在右侧列表里找到目标授权码。
2. 点 `编辑`。
3. 把 `会员模式` 改成 `会员 / 托管网关`。
4. 套餐填 `monthly`。
5. 补齐网关 URL、Token、模型和配额。
6. 点 `保存修改`。

### 批量修改

适合“同一批发出的老码一起改成月卡”的场景。

1. 勾选多个授权码。
2. 点 `批量改月卡`。
3. 统一填写：
   - 到期日期
   - 套餐
   - 会员模式
   - 网关 URL
   - Token
   - 默认模型
   - 可用模型
   - 配额 JSON
4. 点 `批量保存`。

### 修改后会发生什么

客户端下次刷新会员状态，或者重启后，会重新拿到最新权限。

历史激活不会丢。

## 6. 如何确认改动已经生效

### 后台侧

看列表里这几个字段：

- 是否显示 `会员`
- 是否显示正确的 `套餐`
- 网关地址是否正确
- Token 是否已配置

### 客户端侧

在启动器里做：

1. 刷新会员状态。
2. 重启启动器。
3. 再看环境诊断和会员信息。

如果会员网关、默认模型、配额都刷新出来，说明生效了。

## 6.1 如何查客户和解绑设备

授权码列表上方可以直接搜索：

- 客户名称
- 授权码完整码
- 授权码尾号

也可以筛选：

- 启用 / 停用
- 已激活 / 未激活
- 会员 / 普通授权

列表默认每页 50 条。客户需要换机器时，点激活次数旁边的 `详情`，在弹窗里确认 `installId` 和 `deviceId`，然后点 `解绑` 删除那一条激活记录。解绑后客户需要重新激活。

## 7. 如何回滚

如果改错了：

1. 打开对应授权码。
2. 把会员模式改回普通授权。
3. 清掉网关配置。
4. 保存修改。

如果是一批改错了：

1. 再次勾选同一批授权码。
2. 用正确模板批量覆盖。

建议在改动前先导出一份 TXT 备份。

当前后台还会在每次数据变更前自动备份数据库。默认目录：

```bash
/opt/openclaw-license/backups
```

操作审计表格里的“备份”列会显示本次变更对应的备份文件名。如果需要回滚整个数据库，先停服务，再用备份覆盖正式库：

```bash
cd /opt/openclaw-license
systemctl stop openclaw-license
cp backups/替换成审计里显示的备份文件.db license.db
systemctl start openclaw-license
systemctl status openclaw-license --no-pager
```

注意：数据库级回滚会回到备份时刻，备份之后新生成的授权码和激活记录也会一起回退。只有单个客户填错时，优先用后台编辑功能修正，不要直接回滚全库。

## 8. 生产环境建议

### 必做

- 每次改动前先导出授权码列表。
- 管理员 Token 定期轮换。
- 数据库定期备份。
- 批量修改、删除、清空后检查“操作审计”，确认备份文件已经生成。
- 只保留一套正式套餐模板。

### 不建议

- 不要在客户交付现场临时改一堆字段。
- 不要把网关 Token 直接发给客户。
- 不要把月卡套餐名随手改成十几个版本。

## 9. 常见问题

### 9.1 客户为什么没拿到新权限？

通常是这几种情况：

- 客户没有刷新会员状态。
- 客户还在用旧缓存。
- 你改的是普通码，没有开启会员模式。
- 网关 Token 没补全。

### 9.2 为什么编辑后看起来没变？

先确认：

- 列表是否已刷新
- 是否保存成功
- 客户端是否重启或刷新

### 9.3 为什么批量修改后还有旧值？

一般是客户端本地还有旧会话缓存。  
让客户端执行一次刷新会员状态，再重启。

### 9.4 为什么后台提示网关未配置？

表示这个码没有完整的：

- `网关 Base URL`
- `网关 Token`

至少要补全这两个字段。

## 11. 备注

这个后台后续如果再增强，优先级建议是：

1. 配额扣量。
2. 多管理员/RBAC。
3. 网关 Token 加密存储。
4. 审计日志筛选和导出。
5. 套餐模板导入/导出。
## 网关字段补充说明
如果你要统一维护套餐里的网关模型，优先把下面三个字段一起填完整：
- `gatewayDefaultModel`：默认模型
- `gatewayImageModel`：图像模型
- `gatewayVideoModel`：视频模型

如果你要把生图、视频分到不同供应商或不同 API Key，补下面两个字段：
- `gatewayImageToken` / `gatewayImageAccessToken`：图像 API Key
- `gatewayVideoToken` / `gatewayVideoAccessToken`：视频 API Key

这两个字段留空时会自动继承通用 `gatewayAccessToken`，旧授权码不需要重新生成也能兼容。

空着的话，客户端拿到的还是普通授权，不会带会员网关能力。
