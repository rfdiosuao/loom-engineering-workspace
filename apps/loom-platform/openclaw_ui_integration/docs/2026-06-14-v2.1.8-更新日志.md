## 重点改动

### 1. 启动器自更新链路加固

- 自更新 manifest 和安装包地址要求使用 HTTPS。
- 安装包域名必须与更新源一致，或显式加入 `OPENCLAW_LAUNCHER_UPDATE_ALLOW_HOSTS` 白名单。
- 更新包必须提供 64 位十六进制 `sha256`，避免下载到未知内容后继续安装。
- 失败会更早暴露为配置问题，不再把高风险更新源当成正常包处理。

### 2. 手机自动化与定时任务更稳定

- `phone_scheduler.py` 改为时区感知时间计算，默认使用 `Asia/Shanghai`。
- 可通过 `OPENCLAW_SCHEDULER_TZ` 指定定时任务时区。
- Python 端任务队列新增 `.rw.lock` 写锁，降低并发写入造成队列损坏的概率。
- Node 端 `openclaw-phone-agent.mjs` 增加队列写锁和合并写入逻辑，避免 drain 旧快照时覆盖新任务。
- 调度接口中的同步文件操作切到线程池执行，避免阻塞 FastAPI 事件循环。

### 3. 桌面 RPA 与跨平台适配

- `process.py` 拆分 Windows 与 POSIX 进程处理逻辑。
- macOS/Linux 侧优先使用 `lsof` 和 `os.kill` 处理端口进程。
- Python runtime 查找逻辑补齐 macOS/Linux 常见路径。
- 移除 `D:\python\python.exe` 这类机器绑定路径，避免便携包在新机器上误判运行环境。
- 这批改动主要是给 Mac 端同步和后续桌面 RPA 稳定化打底。

### 4. 图像/视频网关安全边界

- 新增 `url_safety.py`，统一校验外部媒体 URL。
- 默认拒绝 localhost、内网、链路本地、多播、未指定地址等高风险目标。
- 如确实需要使用内网媒体网关，可显式设置 `OPENCLAW_ALLOW_PRIVATE_MEDIA_URLS=1`。
- 图像和视频服务都接入同一套 URL 安全检查，减少 SSRF 和误配风险。

### 5. CI、文档与仓库清理

- 版本一致性校验同步到 `2.1.8`。
- 手机自动化、调度、性能验证脚本已纳入当前验证口径。
- VitePress 文档站和 `openclaw-cli.skill` 已进入仓库同步范围。
- `.gitignore` 补齐本地 scratch、日志、截图帧、录屏、预览产物、临时发布包等目录，减少误提交。

### 6. AI 视频生成稳定性与等待体验

- 视频生成改为提交 job 后持久化当前 `jobId`，切换到其它模块再回来仍会恢复同一个生成任务的进度动画。
- 视频工作台拆分图像 busy 与视频 busy，避免视频生成时把图像预览区也误显示成加载状态。
- 视频生成等待卡显示真实已用时，恢复任务时不再从 0 秒重新计时。
- Agnes、Seedance、DashScope 轮询默认上限统一提升到 20 分钟。
- 新增 `OPENCLAW_VIDEO_POLL_TIMEOUT_SEC`，可按部署环境调整视频任务最大等待时间。
- 视频任务提交阶段增加瞬时错误重试，覆盖 `429/5xx`、网关断流、SSL EOF、连接重置等临时失败。
- 视频任务下载阶段增加瞬时错误重试，减少临时 CDN/网关断流导致的失败。
- 视频轮询状态文案产品化，不再直接把 `queued`、`in_progress` 等原始状态抛给用户。
- Agnes 真实网关使用 `agnes-video-v2.0` 做连续生成压测，验证提交、轮询、下载和 MP4 文件头。
- 本轮 Agnes 原始压测 10 次：9 次成功，1 次 SSL EOF；成功任务平均 `157.3s`，最快 `67.0s`，最慢 `266.8s`。
- 使用授权码标签 `NBR8-PGSQ` 做模型 smoke：`agnes-2.0-flash`、`agnes-image-2.1-flash`、`agnes-video-v2.0` 均通过。

## 用户能感知到的变化

- 启动器更新包配置错误会更明确地失败，而不是下载后再出现不透明异常。
- 手机定时任务更不容易出现丢任务、重复覆盖、并发写坏队列的问题。
- Mac 端同步源码时，遇到 Windows 专属命令或硬编码路径的概率降低。
- 图像/视频生成如果传入内网 URL，默认会被拦截；这是安全策略，不是服务宕机。
- 视频生成不再因为用户切到别的模块而丢失等待体感，回来仍能看到生成中的状态。
- 仓库状态更干净，后续 Mac 端、CI/CD 和发版同步更容易判断真实改动。

## 兼容性与配置注意事项

- 自更新 manifest 必须提供合法 `sha256`。
- 自更新安装包地址建议与 manifest 同源；跨域 CDN 需要配置 `OPENCLAW_LAUNCHER_UPDATE_ALLOW_HOSTS`。
- 定时任务默认按 `Asia/Shanghai` 执行；海外服务器或 Mac 端可设置 `OPENCLAW_SCHEDULER_TZ`。
- 视频生成默认最多轮询 20 分钟；如网关排队更久，可设置 `OPENCLAW_VIDEO_POLL_TIMEOUT_SEC`。
- 内网媒体 URL 默认不允许访问；只有确认风险后才设置 `OPENCLAW_ALLOW_PRIVATE_MEDIA_URLS=1`。
- 完整的 Tauri updater 签名、Windows Authenticode、macOS notarization 仍应作为后续正式发版项推进。

## 已知未完成

- 本次提交是源码稳定化和打包前置修复，不等同于已经完成 Windows/Mac 安装包签名发布。
- 自更新链路目前已强化校验，但还没有完成完整 Ed25519 签名更新体系。
- 真机手机自动化仍建议做长时间 soak test，尤其是咸鱼定时发布、一键擦亮、曝光类任务。
- 桌面 RPA 的启动/停止延迟还需要继续做性能剖析，当前版本先解决跨平台和路径硬伤。

## 后续建议

- 先让 GitHub CI/CD 跑完整构建，再生成 Windows 离线包、在线便携包和 Mac 端源码同步包。
- Mac 端优先验证 Python runtime 查找、端口进程清理、桌面 RPA 安装入口。
- 手机自动化任务库发布前，应保留任务日志、截图证据和失败重试策略，避免只看按钮能点就算完成。
