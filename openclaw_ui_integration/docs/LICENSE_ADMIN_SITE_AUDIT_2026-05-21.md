# 授权后台线上站点问题记录

> 检查对象：`https://license.heang.top/admin`  
> 检查时间：2026-05-21  
> 检查方式：线上 HTML 抓取、Edge Headless 桌面/移动截图、公开接口状态、响应头检查。  
> 结论：线上已经是新版后台，但还有几项生产体验和安全治理债需要排期。

## 已确认正常

- 线上 HTML 已包含新版入口：`套餐模板`、`操作审计`、`激活详情`、`搜索客户 / 授权码`。
- `/health` 返回 200。
- 未带管理员 Token 访问 `/admin/api/codes`、`/admin/api/plans`、`/admin/api/audit-logs` 返回 401。
- `X-Frame-Options: SAMEORIGIN` 和 `X-Content-Type-Options: nosniff` 已存在。

## 已修复：左侧导航无响应

证据：`会员套餐`、`网关配置`、`操作审计`、`文档` 原来只是静态按钮，没有绑定点击事件。

修复：

- 左侧导航已改成真实锚点导航。
- `会员套餐` 跳转到套餐模板区。
- `网关配置` 跳转到独立网关配置区，并展示各套餐网关是否已配置。
- `操作审计` 跳转到审计表。
- `文档` 跳转到文档区，列出操作手册、生产债务台账、站点审计记录。

验收方式：

- 点击左侧五个导航按钮，页面应滚动到对应区域。
- 当前导航按钮应高亮。

## 已修复：移动端布局溢出

证据：390px 宽移动截图中，`授权中心` 顶部按钮区域被挤到右侧，`刷新` 按钮只露出一部分。

影响：手机或窄屏远程桌面里无法完整操作顶部按钮。

修复：

- 在 `@media (max-width: 700px)` 下让 `.topbar` 改成纵向布局。
- `.top-actions` 设为 `width: 100%`，按钮改成两列排列。
- 给 `.content` 设置 `min-width: 0`，避免内部 flex 子元素撑开页面。

验收方式：

- 390 x 844 截图里不出现横向裁切。
- `刷新`、`导出全部`、`批量改月卡`、`清空全部` 全部可见。

## P1：安全响应头不完整

证据：线上 `/admin` 缺少：

- `Content-Security-Policy`
- `Referrer-Policy`
- `Strict-Transport-Security`
- `Cache-Control`

影响：后台页面抗注入、跳转泄露、HTTPS 强制和敏感页面缓存控制不足。

建议修复：

- 在 Nginx 或 Python 服务层补安全头。
- `/admin` 和 `/admin/api/*` 至少设置 `Cache-Control: no-store`。
- HTTPS 正常后补 `Strict-Transport-Security: max-age=31536000; includeSubDomains`。

## P1：管理员 Token 存在浏览器 localStorage

证据：后台前端使用 `localStorage.setItem("openclawAdminToken", token())`。

影响：如果浏览器被同源脚本污染或电脑被临时借用，Token 更容易长期残留。

建议修复：

- 短期：增加“退出登录/清除 Token”按钮。
- 中期：改成服务端 session cookie，`HttpOnly + Secure + SameSite=Strict`。
- 至少给后台文档写明：公用电脑不要保存 Token。

## P1：清空全部仍是浏览器 confirm

证据：`清空全部`、`删除授权码`、`解绑激活` 都依赖 `confirm()`。

影响：高危操作容易误点；浏览器原生确认框不展示足够上下文，也不要求二次输入。

建议修复：

- `清空全部` 改成自定义弹窗，要求输入 `CLEAR`。
- 删除/解绑弹窗展示客户、授权码尾号、设备 ID。
- 后端可以增加 `confirmText` 校验，防止绕过前端。

## P1：未登录态信息层级不够明确

证据：未输入 Token 时，套餐模板、授权列表、审计表各自显示“请输入管理员 Token 后刷新”，但页面顶部仍显示完整操作按钮。

影响：新用户可能以为服务异常或按钮可直接操作。

建议修复：

- 未登录时顶部显示统一登录态提示。
- 禁用 `刷新` 以外的危险按钮：`导出全部`、`批量改月卡`、`清空全部`。
- Token 保存后自动刷新所有面板。

## P2：审计检索能力不足

证据：当前审计表只展示最近记录，没有筛选、导出、按授权码定位。

影响：后续生产数据多了以后，售后追查会变慢。

建议修复：

- 审计按动作、目标授权码、时间范围筛选。
- 支持导出 CSV。
- 授权码详情页显示该码相关审计记录。

## 后续建议

优先先修 P0 移动端布局，再补 P1 安全头和 Token 退出登录。审计筛选、CSV 导出可以放到后续运营增强。
