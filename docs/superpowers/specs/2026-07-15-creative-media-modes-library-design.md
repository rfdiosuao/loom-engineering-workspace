# 麓鸣图生图、图生视频与本机素材库设计

**状态：** 用户已确认交互方向，等待书面规格复核

**日期：** 2026-07-15

**范围：** `openclaw_new_launcher` 创作页、媒体 Bridge API、LOOM CLI 媒体输出、测试与受保护 NSIS 发布流程

## 1. 背景

当前麓鸣“生图 / 生视频”页面已经具备文生图、文生视频、异步任务轮询、本地文件落盘和结果预览能力。Bridge 也已经接受图生图所需的 `editImagePath`，视频服务已经接受 `mode: i2v` 与 `imagePath`。现状缺少的是用户可见的参考图入口、图生模式校验、清晰的常用比例选择，以及跨应用重启可恢复的素材展示。

现有生成接口会把图片写入 `data/generated-images`，把视频写入 `data/videos`。`loom_cli.py media image/video` 通过同一 Bridge 调用这些接口；`npm run phone:image` 默认也写入 `data/generated-images`。文件已经落盘，但页面只持有当前任务返回的 Base64，重启页面后不会重新读取，因此用户感知上仍像“结果丢失”。

本次改动把现有能力接成完整产品闭环，不重写模型客户端，不引入云存储，也不增加自动发布或外部触达能力。

## 2. 目标

1. 在图片和视频两个现有主入口内分别提供文生与图生模式。
2. 图生图、图生视频同时支持本地上传参考图和复用刚生成的图片。
3. 图片只展示六个常用比例：`1:1`、`3:4`、`4:3`、`9:16`、`16:9`、`5:2`。
4. 视频继续提供 `16:9`、`9:16`、`1:1` 常用比例。
5. UI 与 CLI 生成的图片、视频在本机长期保存，并在应用重启后继续显示。
6. 历史素材可以预览、复制路径、打开所在位置、继续作为参考图和安全删除。
7. 历史视频通过文件流播放，不把全部视频 Base64 加载到前端内存。
8. 最终生成标准 Tauri NSIS 安装包，并验证发布资源使用受保护构建产物、无 sourcemap、无 TypeScript/TSX 开发源码。

## 3. 非目标

- 不建设云端素材库，不做多电脑同步。
- 不修改图像或视频供应商协议，不承诺所有模型都支持全部比例。
- 不增加自定义宽高输入，不在页面展示复杂像素参数。
- 不增加批量发布、自动评论、自动私信、自动加好友或其他外部触达能力。
- 不迁移或删除现有 `data/generated-images`、`data/videos` 文件。
- 不把素材放入安装包；用户数据继续位于安装目录的受保护 `data` 目录，并沿用无损升级规则。
- 不把字节码包装描述为不可逆加密。发布保护目标是避免直接分发开发源码并提高复制成本，不替代服务端授权和签名校验。

## 4. 页面信息架构

页面继续保留“图片 / 视频”两个主入口，避免四个并列大按钮造成拥挤。

### 4.1 图片入口

图片工作区顶部使用紧凑分段控件：

- `文生图`
- `图生图`

比例控件只展示六个有形状提示的常用比例，不显示像素值：

- `1:1`
- `3:4`
- `4:3`
- `9:16`
- `16:9`
- `5:2`

比例值由一个集中定义的 preset 映射成供应商请求所需的 `size`。映射值不得散落在 JSX 和 Bridge 中；测试必须逐项锁定比例与请求值。供应商拒绝某个尺寸时，页面显示其真实错误，不静默回退到其他比例。

### 4.2 视频入口

视频工作区顶部使用：

- `文生视频`
- `图生视频`

视频比例继续展示 `16:9`、`9:16`、`1:1`。图生视频提交 `mode: "i2v"` 和 `imagePath`；文生视频提交 `mode: "t2v"`，且不得携带残留参考图参数。

### 4.3 参考图区域

参考图区域只在图生模式出现，包含：

- 本地上传按钮，接受 PNG、JPEG、WebP。
- “使用最近生成图片”按钮；没有历史图片时禁用并解释原因。
- 当前参考图预览、文件名或素材来源。
- 替换和移除操作。

本地上传在前端校验 MIME、扩展名和大小后转换为 Data URL，沿用 Bridge 现有临时文件转换能力。上传原图不会自动进入素材库；只有模型生成的输出才进入素材库。切回文生模式时参考图状态保留在当前页面会话中，但请求不得携带该状态。

素材卡上的“图生图”和“图生视频”命令会设置该文件为参考图、切换到对应主入口和模式，并把工作区滚动到参考图与提示词区域。操作只准备任务，不自动提交。

### 4.4 提交校验

- 提示词为空时不提交。
- 图生模式未选择参考图时不提交，主按钮显示明确的不可用原因。
- 不支持的文件类型、读取失败或超出限制时立即提示，Data URL 不进入 React 持久状态之外的存储。
- 图片、视频任务继续使用独立 Job 和独立轮询；一个任务运行时不得错误禁用另一类任务。
- 请求失败保留提示词和参考图，方便用户修正后重试。

## 5. 本机素材库

### 5.1 文件是事实来源

素材库不以单个 JSON 索引作为唯一事实来源。Bridge 每次分页查询时扫描允许目录并构建有界结果：

- 图片：`data/generated-images`
- 视频：`data/videos`

这样可以自动发现三类产物：

1. 页面通过异步生成接口产生的文件。
2. `loom_cli.py media image/video` 产生的文件。
3. `npm run phone:image` 产生的文件。

已有历史文件无需迁移即可显示。可选 sidecar 元数据用于保存提示词、模式、比例、模型和来源；sidecar 缺失或损坏时仍从文件名、扩展名、大小和修改时间生成最小素材记录。

### 5.2 素材记录

公开给前端的素材记录包含：

- 稳定的 opaque `id`
- `kind`: `image | video`
- 文件名、MIME、字节数和创建时间
- 可选比例、提示词、模型、生成模式和来源
- 内容 URL 与缩略图 URL
- 是否可以作为参考图

素材 ID 由类型和规范化相对路径生成摘要。前端不能提交任意绝对路径访问文件。Bridge 对列表、内容、打开位置和删除操作都重新校验文件仍位于允许目录中。

### 5.3 API

新增并保持 localhost Bridge 鉴权：

- `GET /api/media/assets?kind=image|video&cursor=<opaque>&limit=<n>`
- `GET /api/media/assets/{assetId}/content`
- `POST /api/media/assets/{assetId}/reveal`
- `DELETE /api/media/assets/{assetId}`

列表默认返回最近 20 条并支持加载更多。视频内容接口支持 HTTP Range，避免整段视频进入内存。删除必须由 UI 二次确认，只删除所选媒体文件及其同名 sidecar；不得递归删除目录。

### 5.4 结果区域

现有结果区域改为两个层次：

- “本次结果”：保留当前 Job 的即时结果和错误。
- “本机素材库”：显示持久历史，支持图片/视频筛选和加载更多。

素材库读取失败时只影响历史区域，不得清空本次结果或阻止新任务提交。空状态必须区分“没有历史素材”和“素材库读取失败”。

## 6. 数据流

### 6.1 图生图

1. 用户选择图生图。
2. 用户上传本地图片或选择素材库图片。
3. 前端提交提示词、比例映射后的 `size` 和 `editImagePath`。
4. Bridge 将 Data URL 转为临时文件，或验证素材库文件路径。
5. 图像客户端执行编辑。
6. 输出写入 `data/generated-images`，生成可选 sidecar。
7. Job 完成后刷新本次结果和素材库第一页。

### 6.2 图生视频

1. 用户选择图生视频和参考图。
2. 前端提交 `mode: "i2v"`、`imagePath`、提示词、比例、清晰度和秒数。
3. Bridge 将 Data URL 转为临时文件，或验证素材库文件路径。
4. 视频客户端执行现有 i2v 请求。
5. 输出写入 `data/videos`，生成可选 sidecar。
6. Job 完成后刷新本次结果和素材库第一页。

### 6.3 CLI 汇入

CLI 不需要调用新的登记命令。只要输出位于上述允许目录，下一次素材列表查询就能发现。异步 CLI Job 完成时由 Bridge 写入 sidecar；直接 Node 图片 CLI 没有 sidecar时以文件扫描结果显示，并通过文件名前缀标记为 CLI 来源。

## 7. 安全与隐私

- API Key 继续只保存在本机媒体配置，不在素材 sidecar 中记录。
- 本地上传的 Data URL 不写日志、不写配置、不返回到素材列表。
- 素材内容接口只服务允许扩展名与允许目录，拒绝 `..`、符号链接越界和伪造 ID。
- 删除接口拒绝目录、未知文件、允许目录之外的路径和 sidecar 之外的伴随文件。
- 提示词属于本机素材元数据，默认不上传云端；复制或导出由用户主动触发。
- 素材 API 继续受现有 Bridge 会话鉴权与商业功能保护约束。

## 8. 发布保护与 NSIS

当前 Vite 生产构建会压缩前端并且不生成 sourcemap，但 Tauri 配置仍直接收集 `python/**/*` 和 `scripts/**/*`。本次发布不得把“Vite 已压缩”误报为整包混淆。

新增受保护发布入口，要求：

1. 前端使用生产构建、关闭 sourcemap，并扫描安装资源中不存在 `.ts`、`.tsx`、`.map`。
2. Python 发布资源从隔离 staging 目录生成，至少采用无源码字节码形式；仅保留启动所需的最小加载器。staging 必须通过 bundled Python 的 Bridge 启动与 CLI smoke test。
3. Node CLI 发布资源从隔离 staging 目录生成压缩/混淆产物，并对 `loom:image`、手机状态和上下文命令执行 smoke test。
4. 开发目录不得被原地删除、改名或覆盖；保护构建失败必须清理 staging 并保留源码工作区。
5. Tauri NSIS 只能引用通过验证的 staging 资源；标准安装包仍不得捆绑 Codex、Claude Code、opencode、OpenClaw、Hermes 或 APKClaw payload。
6. 最终报告必须区分“前端压缩”“Python 无源码字节码”“Node 压缩/混淆”的实际保护级别，不宣称不可逆。

如果无源码 Python 或 Node 保护产物无法通过真实启动测试，则发布被视为失败，不得回退为携带原始源码后仍标注“已混淆”。

## 9. 错误处理

- 素材目录不存在：创建目录并返回空列表。
- 单个文件损坏：跳过该文件并记录脱敏诊断，不让整个列表失败。
- 参考图在提交前被删除：返回稳定的“参考图不存在”错误并保留页面输入。
- 视频服务不支持 i2v：显示供应商真实错误与当前模型，不自动改为文生视频。
- 比例被供应商拒绝：显示不支持当前比例，不自动换比例。
- Range 非法：返回标准 416，不读取整个视频。
- 删除失败：保留素材卡并提示占用或权限原因。
- 素材刷新失败：保留上一次成功列表并显示“刷新失败”。

## 10. 测试与验收

### 10.1 前端

- 契约测试要求存在图片/视频主入口和四个模式标记。
- 验证六个图片比例只显示比例，不显示自定义宽高。
- 验证文生请求不携带参考图，图生请求携带正确字段。
- 验证缺少参考图、错误文件类型和超限文件不会提交。
- 验证素材卡可以准备图生图和图生视频，但不会自动提交。
- 验证图片与视频 Job 仍相互独立。

### 10.2 Bridge 与 CLI

- 临时目录构造 UI、Python CLI 和 Node CLI 命名的图片，验证均被扫描。
- 验证分页稳定、按时间倒序、损坏 sidecar 可恢复和重复路径去重。
- 验证内容读取、视频 Range、打开位置和安全删除。
- 验证路径穿越、符号链接越界、未知 ID 和目录删除被拒绝。
- 验证图生图 Data URL 临时文件在成功与失败后都清理。
- 验证图生视频真实传递 `i2v` 和参考图。

### 10.3 发布

- `npm run build`
- `python -m unittest discover -s python/tests -p "test_*.py"`
- Node 平台契约与现有 CLI 测试
- 受保护资源 staging smoke test
- `npm run tauri -- build --bundles nsis` 的受保护配置
- NSIS 内容扫描：无开发 TS/TSX/sourcemap、无原始受保护 Python/Node 源码、无第三方 Agent payload
- 安装、启动、素材跨重启可见、旧版本覆盖升级和卸载 smoke test

## 11. 回滚

代码回滚时只撤销模式 UI、素材 API 和受保护发布配置，不删除 `data/generated-images`、`data/videos` 或 sidecar。旧版本仍可继续使用原有文生图、文生视频接口；新增素材文件保持普通 PNG/JPEG/WebP/MP4，可由用户直接访问。

## 12. 完成条件

1. 文生图、图生图、文生视频、图生视频都可从现有创作页提交真实任务。
2. 图生模式可使用本地上传和刚生成图片。
3. 图片页面只出现确认的六个常用比例，并包含 `5:2`。
4. UI 与 CLI 生成素材在重启后仍显示，历史视频不会造成页面内存激增。
5. 文件访问和删除通过路径安全测试。
6. 完整构建、Python、Node、契约和发布 smoke test 通过。
7. 生成标准 Tauri NSIS 安装包，报告路径、SHA256、保护级别和内容扫描结果。
