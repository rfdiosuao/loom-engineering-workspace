# LOOM 与 APKClaw 技术债台账

> 更新日期：2026-07-16
>
> 基线版本：2.1.89
>
> 原则：只记录仍需推进的债务；已解决事项保留证据，不重复立项。

## 已关闭

### 安装包内置 Python

- 内置 Python 同时支持 Bridge 使用的 `core.*` 和标准测试使用的 `python.*` 导入。
- 构建脚本会执行两类导入自检。
- CI 会重新生成内置 Python，并使用它执行完整测试。

### 手机凭据落盘

- 手机令牌和配对密钥使用当前 Windows 用户的 DPAPI 加密。
- 旧版明文配置在首次读取时自动迁移。
- 前端和公开 API 只返回凭据是否可用，不返回原文。

### 模板云端上传

- 默认仅本地保存。
- 用户显式同意或管理部署设置环境变量后才允许上传。
- 同意、关闭和重试都有 API 与 CLI 契约测试。

### 飞书“假同步”

- 写入命令成功后必须按 `recordId` 回读。
- 远端字段一致后才标记 `synced`。
- 回读失败进入待核验队列，可通过 reconcile 重试。

### 飞书幂等与并发导入

- 线索使用 `dedupeKey` 作为稳定业务键，并复用现有“任务日志 ID”字段，无需迁移表结构。
- 兼容原有 `logId/taskLogId` 作为业务键，既有 CLI 和测试写入行为不漂移。
- 写入前精确查询：唯一记录执行更新，重复业务键停止写入并报告冲突。
- 写入响应缺少 `recordId` 时按业务键反查，不会直接再次创建。
- 并发线索导入纳入矩阵状态事务锁，相同线索只创建一组本地记录并执行一次飞书写入。
- 待同步队列保留稳定业务键和完整远端字段，重试不会退回 `leadId`、丢失字段或因单条异常清空整批队列。
- 单 Bridge 内飞书写入使用可重入事务锁，同业务键并发同步只创建一条远端记录。
- 飞书 CLI 超时和运行异常转为 `sync_failed` 待同步记录，本地线索仍会落盘。

### Bridge 兼容性识别

- `/api/system/info` 返回启动器版本、API 契约版本和能力清单。
- 安装包运行时优先读取 Tauri 注入的真实应用版本。

## P0：发布前必须完成

### 1. 签名安装包闭环

- 在 GitHub Windows Runner 配置正式代码签名证书。
- 生成受保护 NSIS 安装包并验证 Authenticode 签名、SHA256 和版本号。
- 在已安装旧版、正在运行 Bridge、Node、Codex 的机器上验证覆盖升级和进程退出。

验收证据：CI 链接、签名校验输出、安装/升级/卸载录像或截图。

### 2. 三台真实手机稳定性

- 三台不同 Android 设备并发下发只读任务。
- 验证一台断网、一台令牌失效、一台正常时互不拖累。
- 连续运行至少两小时，检查状态、忙闲、日志、内存和重连。
- 所有发布、评论、私信、加好友动作继续要求人工确认、白名单、频控和审计。

验收证据：任务 ID、设备 ID、事件流、失败恢复和资源曲线。

### 3. 真实飞书多维表格

- 新账号登录、新建表、绑定已有表各跑一次。
- 写入一条真实测试线索并回读核验。
- 模拟断网后重试，确认不会重复创建客户记录。
- 检查字段改名、删除、只读公式字段时的错误提示。

验收证据：脱敏表格截图、recordId、reconcile 输出和本地审计日志。

## P1：稳定版后的首轮迭代

### 1. Matrix 与 APKClaw 长连接契约

- 手机端将 `progressLog` 作为兼容字段正式写入任务状态和 SSE。
- Bridge 不再依靠 `round + toolId` 猜测全部中文进度。
- 增加契约版本和旧 APK 兼容测试。

### 2. 更新器真实故障矩阵

- 覆盖文件占用、杀毒软件拦截、磁盘不足、网络中断、旧版残留和中文用户目录。
- 失败时保留旧版本可启动状态，并提供明确恢复动作。

## P2：维护性重构

### 1. 超大路由文件拆分

- `routes_phone.py` 按配置、任务、事件流、截图和模型同步拆分。
- `routes_matrix.py` 按调度、获客、模板、飞书集成拆分。
- 保持现有 HTTP 与 CLI 契约不变，分批迁移。

### 2. 前后端共享能力清单

- 当前能力名由 Bridge 返回，前端已有类型声明。
- 后续生成统一 schema，减少 Python/TypeScript 手工同步。

### 3. 文档与发布证据归档

- 每个稳定版保存更新说明、测试数量、CI 链接、签名摘要和真实设备验收结论。
- 不把本地日志、客户数据、账号令牌或私钥提交到仓库。
