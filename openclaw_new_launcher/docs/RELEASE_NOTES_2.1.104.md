# LOOM 2.1.104 更新说明

## 智能体能力

- 原生智能体完整继承手机控制、手机矩阵、生图、生视频、素材传输和系统状态能力。
- 单台在线手机可自动绑定到当前任务；多台设备目标不明确时仍要求明确选择，避免任务发错手机。
- 后续发布任务可复用当前会话已经生成的图片和视频，不再无意义地重复生成。
- 工具调用、目标缺失和关键操作错误统一显示可执行的中文提示，不再暴露内部能力 ID。

## 长任务稳定性

- 修复“深度 + 完全控制”任务仍在手机执行，但 LOOM 提前显示超时和设备异常的问题。
- 深度任务单次读屏与状态轮询预算由 20 秒提高到 30 秒，覆盖慢设备的真实截图耗时。
- 临时网络超时或手机状态接口 5xx 时继续追踪原 `taskId`，不会误判整单失败或重复下发任务。
- 达到总任务时限时保留“手机可能仍在执行”的结构化诊断信息，便于后续恢复和排查。

## 矩阵截图性能

- 新增多设备批量截图接口，减少手机数量增加时的 HTTP 请求和前端连接竞争。
- 聚焦手机保持 960px 清晰帧，后台手机使用 640px 轻量帧；手动截图仍自动切换为高清。
- 后端截图并发限制为 6 路，避免大量 Node 子进程同时占满 CPU、局域网和手机端服务。
- 修复并发截图共用秒级文件名导致画面互相覆盖的问题。
- 截图缓存按“设备 + 图像规格”隔离，杜绝不同手机之间串图，并持久化尺寸与屏幕哈希元数据。
